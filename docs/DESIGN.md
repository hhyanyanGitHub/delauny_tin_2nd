# 设计与后续路线

## 当前结构

```text
调用系统 / GUI
      |
稳定 C ABI + Legacy 包装
      |
Context、错误处理、结果对象
      |
CGAL Delaunay hierarchy ---- Boost face R-tree
      |                         |
顶点 ID / Z                 范围相交查询
      |
DTIN / DTMESH / XYZ 持久化
      |
GRID / Contour / Async Task（v0.3）
      |
可选 GDAL Adapter + CRS WKT（v0.4）
      |
独立 Constrained Delaunay + Domain Marking（v0.7）
```

核心是 2.5D 数据模型：CGAL 顶点位置只保存 XY，自定义顶点信息保存 Z 和稳定 ID。
面不复制 XYZ。R-tree 保存面 XY 包围盒和 CGAL 面句柄；编辑前删除旧面索引项，
编辑后只插入新增面。

v0.3 在 TIN 上方增加独立 `Grid` 和 `ContourSet` 数据对象。三者没有继承关系，
通过显式转换函数连接，避免把“等高线反推地形”误表示为无损类型转换。耗时转换
可由任务对象持有源数据共享生命周期，并通过协作取消停止。

v0.4 将 CRS 作为 TIN、GRID、ContourSet 的 UTF-8 WKT 元数据保存，转换时传播，
不参与几何计算。GDAL 适配器位于核心数据对象之外；关闭 `DT_WITH_GDAL` 时保留
相同 C ABI 导出并返回 `DT_E_UNSUPPORTED`。GeoTIFF 角点 geotransform 与本库节点
geotransform 在边界处做半像元中心换算。COG 使用 GDAL `CreateCopy`，GeoPackage
等高线使用 `LineStringZ + elevation + closed` 模式。

v0.13 将 GDAL 交换适配层接入 Win32 GUI。程序初始化时分别探测 `GTiff`、`COG`
和 `GPKG` 驱动，仅启用当前环境真正可用的菜单。GeoTIFF/COG 导入替换 GRID 句柄，
GeoPackage 导入替换等高线句柄；两条路径都保留普通 TIN、CDT 和无关派生图层，
从而支持叠加对照。导出使用显式创建选项：GTiff 为分块、DEFLATE 和安全 BigTIFF，
COG 为 DEFLATE 和安全 BigTIFF，GeoPackage 建立空间索引。

v0.14 在 GUI 层增加任意直线剖面分析，不扩展 DLL ABI。`terrain_profile.hpp` 将
线性采样、三角形重心插值、六参数仿射 GRID 反算、双线性插值和剖面统计写成
无 Win32 依赖的纯数学函数，由独立测试覆盖。GUI 以 401 个等距节点采样整条 A—B；
数据源在开始计算时一次选定，优先可见 CDT、TIN、GRID，再考虑隐藏但存在的图层，
避免逐样本回退造成曲线来源不一致。CDT 无效域、TIN 凸包外和 GRID NoData 被记录
为断点，统计按连续有效段累计上升、下降和最大绝对坡度。

v0.15 延续“不扩展 DLL ABI”的 GUI 分析层设计。`terrain_measurement.hpp` 提供无
Win32 依赖的简单多边形校验、稳定平移鞋带面积、耳切三角化和数值积分。平面面积、
周长由边界坐标直接计算；每个耳三角形按面积分配细分级数，总预算约 20,000 个微
三角形。共享格点通过与剖面相同的 CDT/TIN/GRID 采样器取高程，微三角形中心用于
判定有效域；中心无效则不计入覆盖，角点无效但中心有效时使用中心高程，避免孔洞或
NoData 边缘产生异常长斜面。有效地表面积、高程统计和相对水平基准面的挖填方因此
属于分辨率受控的数值估算，而不是解析裁剪或工程结算模型。

v0.7 增加独立 `CdtContext`，使用 CGAL
`Constrained_Delaunay_triangulation_2`。普通 TIN 保持原层次 Delaunay 与 R-tree
路径，避免约束能力改变既有动态编辑性能和 ABI。CDT 将断裂线与域边界分开：断裂线
只强制成边；外边界/孔洞边被记录为域屏障，从无限面广度优先遍历，每跨一次屏障
嵌套层级加一，奇数层为域内。

约束增删会复制基础点和约束记录，在候选 CDT 中完成插入、域标记和统计后再交换
状态，因此交叉约束或坏数据不会留下半成品。当前未分段交叉必须由调用方先加入共享
交点。DCDT 文本保存基础点、约束稳定 ID、类型、闭合标志、XYZ 和 CRS。

## 并发

每个上下文具有读写锁。当前安装的 GCC 14 MinGW 与 CGAL 6.0.1 的非平凡 TLS ABI
不兼容，因此 MinGW 构建定义 `CGAL_HAS_NO_THREADS`，并在库内用全局递归互斥锁
串行保护 CGAL 调用。MSVC/更新 CGAL 工具链验证后可恢复不同上下文的并行查询。

## 已知限制

- CDT 已支持断裂线、外边界、孔洞和稳定 ID 更新；单项约束增删改当前仍为全量重建，批量事务只重建一次，但尚未局部化；
- 未分段的交叉约束当前拒绝，调用方需在交点处分段；
- GRID 当前为内存连续 `double` 数组；GDAL I/O 分行访问，但句柄本身尚未外存化；
- GRID→TIN 仍是普通 Delaunay，NoData 空洞只能拒绝或由调用方显式允许桥接；
- 当前等高线不输出完全水平平台的边界；
- `.dtin` v1 保存点集并在加载时重建，不保存逐面拓扑；`.dtmesh` 保存显式
  三角形并在打开时逐面验证，但当前仍只接受由这些顶点形成的 Delaunay 拓扑；
- 范围查询结果尚未提供流式游标；
- R-tree 当前为逐面动态索引，尚未实现压缩的块级索引；
- 旧接口使用全局默认上下文并串行执行；
- 凸包外插入的“边界”只返回可绘制的有限边，不表示无穷边。

## GUI 演示

当前已实现无额外框架依赖的 Win32/GDI 2D/3D 演示程序，验证 DLL 集成、视口范围
检索、地形分级着色、编辑影响显示、保存加载及三维漫游。二维视口结果超过45,000
个三角形时等间隔抽样；三维使用模型中心归一化、轨道相机、透视投影、画家算法
深度排序和约18,000面固定渲染预算。相机数学位于 `gui/terrain_3d.hpp`，不依赖
Win32，并由独立单元测试覆盖。

三维视图支持鼠标环视/平移/缩放、键盘水平漫游与升降、0.5～8倍垂直夸张和相机
适屏复位。二维编辑仍为主工作视图，选择查询、插入、删除或框选放大时自动返回
二维，避免把透视屏幕坐标误当作地形 XY。

v0.6 在同一 GUI 中增加 TIN、GRID、等高线三类图层状态。v0.26 已把原先最多缓存
2000万个双精度值的路径替换为 `dt_grid_read_overview()`：显示层只持有不超过
512×512 的 LOD 位图，再用 GDI `PlgBlt` 将源矩形映射到六参数仿射变换的三个世界
角点；因此超大、旋转或错切 GRID 都不需要整幅 GUI 副本。v0.27 又用
`dt_grid_get_view_window()` 在缩放和平移后只读取视口相交源窗口，使局部放大获得新的
512×512 细节而不是放大全图缓存。等高线直接借用句柄的
稳定点数组，按约20万显示顶点预算抽样，不复制完整线集。

转换菜单调用既有稳定 C ABI：TIN→GRID 自动生成最长边401节点的规则网，等高距
按高程范围取 `1/2/5×10ⁿ` 级数。图层范围参与二维全图计算。TIN 重建或动态编辑
会销毁过期 GRID/等高线；GRID→TIN 保留源 GRID，但清除过期等高线。DGRID 与
DCONTOUR 文本可独立导入导出并与 TIN 叠加。

v0.13 的“数据交换”菜单进一步开放 GeoTIFF/COG GRID 与 GeoPackage 等高线往返。
无 GDAL 构建仍使用同一 GUI 源码，对应命令置灰；启用 GDAL 的构建按具体驱动粒度
启用命令。文件操作结束后重新生成显示缓存、联合范围和状态栏统计，导入失败时不
替换原图层。

v0.14 的“分析”菜单提供两次单击剖面、清除结果和 CSV 导出。青色 A—B 线与端点
叠加在地图上，底部面板显示有效连续段的高程曲线和汇总统计。CDT/TIN 通过既有

1×1/2×2 窗口完成双线性插值，所以千万级 GRID 不需要复制完整节点数组。源图层

v0.12 补充等高线→TIN/GRID。核心先按可选最大段长采样，以精确 XY 或容差网格

## 统一地表微分分析

v0.16 在稳定 C ABI 中引入 `dt_surface_analysis`，由普通 TIN、CDT 和 GRID 三种
句柄共同输出。公共纯数学层位于 `src/dt_surface_analysis.hpp`：输入查询点高程和
世界坐标梯度，统一生成坡度、下坡坡向与向上单位法向，避免三个后端产生角度约定
差异。查询接口只写调用方结构，不产生结果句柄或跨 DLL 内存所有权。

TIN/CDT 后端在现有 CGAL 定位锁内选择有限支撑面，通过平移到首顶点的稳定平面
方程计算 `dz/dx`、`dz/dy` 和查询 Z。CDT 额外要求支撑面属于有效域，孔洞和外域
不参与分析。边/顶点位置可能具有多个不连续邻面，当前契约是确定性选择一个有效
邻面并返回位置标志，而不是对邻面法向求平滑平均；这保留了断裂线两侧的真实分片
线性语义。

GRID 后端先用仿射矩阵逆变换得到连续列/行坐标，在局部 2×2 节点单元求双线性
高程及两方向解析导数，再通过仿射雅可比转为世界 XY 梯度。四个节点任一 NoData
即拒绝该点，避免跨无效区估算。由于双线性单元不是恒定平面，坡度、坡向与法向
随单元内部位置变化，结果通过 `DT_SURFACE_BILINEAR` 明示。

GUI 分析状态保存数据源类型和结构值，不缓存内部面句柄。与剖面和面积量测相同，
源图层被编辑、重建、替换或销毁时只清除依赖该源的坡度结果。二维覆盖层画出支撑
三角形/网格单元、查询点和下坡箭头；数值面板可与面积面板并存。该层不改变三维
网格或任何地形句柄。

## v0.20 全幅地形栅格分析

全幅分析位于 terrain 核心层的 `grid_derive_terrain()`，C ABI
`dt_grid_derive_terrain()` 只负责参数结构兼容、异常到状态码转换和公开句柄所有权。
结果复用 `Grid` 对象而不是引入专用影像类型，因此现有窗口读写、DGRID、GDAL、
CRS 和 NoData 处理无需重复实现，也便于后续把结果继续送入等高线或外部栅格流程。

算法按行扫描源连续数组，输出 GRID 在提交前完全独立。中心差分和边界单边差分均
先在列/行参数空间计算，再使用仿射矩阵行列式转换为世界 XY 梯度；坡度、下坡坡向
与局部查询采用相同角度约定。分析晕渲直接使用向上法向与光源单位向量点积，负值
截断为零。源中心或差分支撑为 NoData 时不做插值，输出 NoData，避免跨越孔洞。

GUI 同时保存原高程 `grid_` 和独立的 `terrain_grid_`。专题句柄只驱动二维预览、
固定量纲色带、图例和专题导出；剖面、面积量测、局部坡度查询及三维地形仍使用原
高程 GRID。切换“恢复高程”只销毁专题句柄并重建预览，不丢失源数据。源 GRID 被
替换或销毁时专题句柄同步释放，避免显示过期派生结果。

## v0.21 异步专题任务与响应式 GUI

`dt_grid_derive_terrain_async()` 复用既有 `dt_task_t` 状态机和 terrain 核心的
`ProgressCallback/CancelCallback`，不复制源 GRID，也不引入新结果类型。任务闭包
持有 `shared_ptr<Grid>`，源公开句柄可在启动后释放；完成结果继续由
`DT_TASK_RESULT_GRID` 和 `dt_task_get_grid_result()` 暴露。进度按完成输出行更新，
取消在行边界检查；v0.22 又在超宽行内部增加周期检查。失败或取消不会发布
半成品 GRID。

GUI 在主线程以 25 ms 有限等待轮询任务，同时分派 Win32 消息并刷新状态栏百分比。
任务运行期间普通菜单命令被门控，`Esc`、显式取消菜单和 `WM_CLOSE` 只设置协作取消
标志；关闭动作延迟到工作线程退出和任务句柄回收后执行，从而避免窗口/句柄生命周期
竞态。z-factor 与光照参数保存在 GUI 会话状态，启动任务时复制进选项结构，工作
线程不访问界面对象。

该版本解决超大 GRID 专题计算冻结消息循环的问题，但算法仍需源、输出各一份连续
double 数组。v0.22 已实现分块并行，v0.26 已消除 GUI 预览的整幅副本；剩余方向是
源 GRID 自身的磁盘映射/流式输出与持久化多分辨率金字塔。

## v0.22 分块并行专题核心

v0.22 将原预留区映射为 `worker_count/tile_row_count`，保持选项结构 80 字节和零值
兼容。行块通过原子递增游标动态领取，可适应 NoData 分布或不同专题公式造成的负载
差异。工作线程直接写新建输出 `Grid::values_` 的互斥行区间；源数组只读，向量大小
在启动线程前固定，因此不存在容器重分配和同元素数据竞争。完成后统一增加一次
generation。串行路径不创建额外线程，便于调试、复现和受限宿主集成。

回调线程模型刻意与计算线程分离：调用线程以条件变量和约 10 ms 周期协调，串行
执行 `ProgressCallback/CancelCallback`；工作线程只做数值计算并通过原子计数报告
完成行数。异常以 `exception_ptr` 捕获，先设置停止标志、汇合全部线程，再在调用线程
重抛；线程创建中途失败也会停止并 join 已创建线程。任何失败或取消都销毁尚未公开
的输出对象。

默认自动线程数取硬件并发度并限制为 32，显式上限为 64，最终还受行块数限制。
64 行默认块高在调度开销与取消响应间折中。4096² 本机基准从 0.233 s 降到
0.0678 s（3.44×），说明该计算已进入明显的内存带宽与 CPU 向量运算混合负载；
不同 CPU、内存通道、GRID 尺寸和专题类型会有不同收益。下一阶段再考虑映射文件、
流式瓦片输出与跨任务共享线程池。

## v0.23 双表面土方核心

v0.23 把原 GUI 水平基准面估算之外的“现状面—设计面”能力下沉为稳定 C ABI。
首版要求两个 GRID 节点对齐、仿射一致且 CRS 相同，刻意不隐式重采样；这样每对
节点有明确高差，NoData 和单位换算也可审计。调用方需在外部完成重投影、对齐和
分辨率选择，避免库静默改变工程量。

每个单元沿固定对角线形成两片三角形，高差在片内线性。全正或全负时用三顶点均值
乘平面面积；符号混合时按边上线性零点解析裁剪正负小三角形，再分别积分。该方法
与微三角形采样预算无关，并保证 `cut-fill` 与带符号积分一致。面积由仿射矩阵行列式
得到，平方高差还使用线性三角形的二次积分闭式公式计算 RMSE。

并行调度沿用 v0.22 的行块思想，但每块统计写入固定索引，完成后按块序归并，从而
避免热点原子浮点累加和调度顺序造成的结果漂移。高差 GRID 的工作线程只写互斥节点
行；最后节点行由协调线程补齐。调用线程串行检查取消和报告进度，异常、线程创建
失败或取消均先汇合工作线程，再销毁未发布结果。

默认整单元 NoData 策略适合工程量覆盖审计；显式半单元标志适合边缘数据利用。
同步接口可不请求高差 GRID，此时除固定大小块统计外不增加全幅输出内存。异步任务
新增 `DT_TASK_RESULT_EARTHWORK`，同时持有两个源句柄，并通过专用 getter 原子发布
统计和可选高差句柄。

GUI 保存独立 `design_grid_`，可加载 DGRID/GeoTIFF 或逐行从现状 GRID 生成偏移
设计面。计算复用现有 25 ms 消息泵和 Esc/关闭取消路径；高差以蓝色填方、白色零差、
红色挖方表达，CSV 保存完整统计。4096² 起伏高差本机基准为串行 0.439 s、自动
并行 0.038 s（11.50×），串并行体积误差为 0。

## v0.24 显式 GRID 重采样核心

v0.24 增加 `grid_resample_like()`，由参考 GRID 完整定义输出的宽高、六参数仿射和
CRS。每个目标节点先映射到世界 XY，再通过源仿射矩阵逆变换得到连续源列/行；实现
因此不区分北向上、旋转、剪切或负像元高。为减少内层循环开销，核心预先把“目标
列/行→源列/行”组合成两个仿射增量，逐行只做加乘而不重复求逆。

最近邻使用连续坐标四舍五入，双线性在源 2×2 节点上插值。默认任一支撑节点为
NoData 就拒绝目标节点；显式归一化标志会忽略无效支撑并按剩余权重重归一，但有效
权重和为零时仍输出 NoData。目标超出源节点包络也写 NoData。源与参考 CRS WKT 必须
相同，接口刻意不调用 GDAL 重投影，避免单位、坐标轴或重采样选择被静默改变。

并行模型复用地形专题的行块领取与协调线程：工作线程只写互斥输出行，协调线程每
约 10 ms 串行报告进度与检查取消；超宽行每 4096 节点检查停止标志。异步任务沿用
`DT_TASK_RESULT_GRID`，同时持有源和参考 GRID，失败或取消不发布半成品。4096²
双线性基准为串行 0.243 s、自动并行 0.0538 s（4.53×），抽样校验和一致。

GUI 加载设计面时只立即拒绝 CRS 不一致；同 CRS 的错位网格被保留，由用户明确选择
双线性或最近邻对齐。对齐任务复用现有响应式消息泵和 Esc/关闭取消路径，取消时原
设计面保持不变；成功后原子替换设计句柄并直接接入 v0.23 土方分析。

## v0.25 GRID 多边形裁剪核心

`grid_clip_polygon()` 接收世界 XY 多边形并用源 GRID 的完整仿射逆矩阵一次性转换到
连续列/行空间。内层循环因此只对整数节点执行包络、边界和偶奇射线测试，不重复作
世界坐标变换。边界节点归入内部，自交边界采用确定性偶奇规则；连续重复点、退化
包络、非法标志和非有限坐标会在分配输出前拒绝。

默认模式保持源宽高和仿射，区域外写 NoData。紧凑模式把多边形连续行列包络向内取
可用整数节点范围并裁到源边界，再把原点平移到首列/首行；列、行向量保持不变。若
包络没有源节点返回 `DT_E_NOT_FOUND`。反向模式保留区域外节点，与紧凑模式互斥。
输出始终带 NoData、继承 CRS，源 NaN/NoData 不会被重新解释为有效高程。

行块并行、协调线程进度、约 10 ms 取消轮询和超宽行停止检查沿用 v0.24 模型。异步
包装在启动线程前深拷贝多边形，因此调用方可立即释放输入数组；任务持有源 GRID，
取消或失败不发布半成品。4096²、8 顶点本机基准为串行 0.378 s、自动并行 0.0755 s
（5.01×），串并行抽样校验和误差为 0。

GUI 复用已完成的简单量测多边形，提供保持范围、紧凑适屏和反向三条命令。任务成功
后原子替换当前 GRID，清除已失效的设计面、专题、土方和等高线缓存，并允许继续
DGRID/GeoTIFF 导出与 GRID→TIN/等高线转换；取消时原 GRID 与选择边界不变。

## v0.26 GRID 窗口概览核心

`Grid::read_overview()` 不创建派生 GRID，而是把指定源行列窗口直接写入调用方缓冲区。
平均、最小和最大方法用整数比例边界把源窗口划分为无缝、互斥的输出分箱；输出不大于
源窗口时，每个源节点恰好访问一次。最近邻方法以分箱中心选择源节点并允许上采样，
适合分类值和环形坡向角。默认聚合忽略 NoData，严格标志则把含任一无效点的分箱整体
置为 NoData。

行块由原子游标动态领取，工作线程只写互不重叠的调用方输出行及对应统计槽。聚合方法
在计算输出的同一次源扫描中累计完整窗口的有效/无效数量、最小、最大和长双精度和；
最后按输出行顺序归并，保证线程调度不改变输出和均值。最近邻统计只覆盖输出样本并由
结果标志明确区分。选项和结果结构均为 64 字节，输出每轴上限 1,048,576，总值数沿用
十亿上限。

GUI 高程层以平均方法生成最多 512×512 的 LOD，并直接使用精确源统计设置色带和等高
距；坡度、阴影和高差也用平均概览。坡向是 0～360° 环形量，算术平均会把 359°/1°
误成 180°，因此专题预览明确使用最近邻。演示层不再保留 `grid_values_` 整幅副本，
超过 2000 万节点的 DGRID/GeoTIFF 也可着色显示并继续从原句柄生成等高线。8192×4096
到 512×512 的基准为串行 0.0427 s、自动并行 0.00963 s（4.43×），结果一致。

## v0.27 世界视口裁剪与自适应 LOD

`Grid::view_window()` 先以六参数仿射的解析逆矩阵将轴对齐世界视口四角映射为源行列
四边形。源节点被视为覆盖半节点边界的矩形域，四边形依次裁过左右上下四条边；空结果
对应 `DT_E_NOT_FOUND`。对剩余顶点取行列极值，再用 `floor/ceil` 和可选 padding 得到
覆盖全部可见节点的最小整数窗口。这个流程比只取逆变换四角的 AABB 更严格：旋转 GRID
与视口完全错开时不会误读一个大窗口，也自然支持剪切和负像元高。

GUI 把高程或当前专题句柄、源窗口和视口有效状态组成轻量缓存键。滚轮与框选立即使键
失效；右/中键拖动时继续显示旧局部位图，松开后只刷新一次，兼顾反馈连续性与扫描成本。
局部位图用所选源窗口四角而非整幅 GRID 四角执行 `PlgBlt`，全图青色外框继续单独绘制。
高程色带仍来自 v0.26 的完整源统计，避免缩放时颜色跳变；等高线、分析和导出也继续
读取完整句柄。

8192×4096→512×512 的全图概览为 0.0128 s，1024×768 局部源窗口为 0.00264 s，约
4.85×；世界视口裁剪约 15 μs。局部源节点占全图约 2.34%，但固定输出着色和线程调度
使耗时不会严格按节点比例缩放。v0.28 已继续实现源 GRID 映射和持久全幅概览。

## v0.28 DGRIDB 映射存储与持久概览

`Grid` 的算法层不区分内存与映射来源。内部 `GridStorage` 提供连续 `double*`、迭代器和
下标语义：普通创建使用 `std::vector<double>`，Windows DGRIDB 加载使用
`CreateFileMapping(PAGE_WRITECOPY)` 与 `MapViewOfFile(FILE_MAP_COPY)`。因此既有地形
分析、等高线、重采样和窗口 LOD 无需维护第二套算法；被修改页面由操作系统私有提交，
源文件保持只读语义。非 Windows 构建保留读入内存的兼容回退。

DGRIDB 固定 64 KiB 头记录尺寸、仿射、NoData、CRS、统计、段偏移、文件大小和头部
FNV-1a 校验。原始节点段按 64 KiB 对齐，便于直接映射；其前保存完整范围最多
512×512 的平均概览。`Grid::read_overview()` 在请求参数完全匹配时直接复制概览，其他
方法和局部窗口仍访问原始节点。节点写入清除旧概览，同时增量维护二进制有效节点数。

保存事务先写同目录唯一临时文件，刷新并关闭后再用 `MoveFileEx(...,
MOVEFILE_REPLACE_EXISTING | MOVEFILE_WRITE_THROUGH)` 替换。若目标就是当前映射源，临时
文件完成后才把私有视图实体化并解除映射，避免 Windows 共享删除限制破坏原子性；
另存路径保持映射，不增加完整驻留副本。8192×4096 基准为保存 0.243 s、打开 6.65 ms、
持久概览 0.177 ms、1024×768 局部读取 3.50 ms。v0.29 已在兼容扩展段中实现多级
金字塔、原始数据块校验与操作系统预取。

## v0.29 DGRIDB 多级金字塔与完整性核心

文件版本仍为 `DGRIDB 1`。固定头 200～255 字节增加扩展魔数、目录/校验表偏移和算法
参数；旧版全幅概览仍紧跟 64 KiB 头，原始节点偏移继续由旧字段 128 指定。金字塔目录
和数据放在两者之间，因此 v0.28 加载器忽略扩展后仍可直接映射原始节点。新加载器把
扩展魔数缺失解释为 v0.28 布局，实现双向读兼容而不引入第二个文件版本。

保存从原始节点生成 1/2、1/4……逐级 2×2 平均，忽略 NoData，直到最长边不超过
512。每级数据按 64 KiB 对齐，Windows 分别以私有写时复制方式映射。概览请求只有设置
`DT_GRID_OVERVIEW_USE_PYRAMID`、方法为平均且非严格 NoData 时才选择层级；选择规则是
“仍不小于输出尺寸的最低分辨率层”。结果设置 `USED_PYRAMID` 而不设置精确源统计位，
所以调用方不会把显示近似误作分析结果。写入节点会同时撤销持久概览、金字塔和校验
能力，下一次保存统一重建。

原始 double 段按 4 MiB 分块计算与文件头一致的 64 位滚动校验。普通加载仅验证固定头、
目录边界和实际文件大小，不扫描原始数据；`dt_grid_verify_binary_file()` 才完整映射并
逐块验证，从而保持打开延迟稳定。`dt_grid_prefetch_window()` 将二维窗口覆盖的最小连续
映射范围交给 Windows `PrefetchVirtualMemory`，函数不可用或数据在普通内存时安全退化
为空操作。

8192×4096 文件为 343.19 MiB，保存 0.594 s、打开 5.17 ms、持久全幅概览 0.107 ms；
4096×2048 局部窗口到 512×256 时，原始精确概览 11.06 ms、金字塔 2.10 ms（5.26×），
完整块校验 0.352 s。v0.30 已实现异步视口请求取消，v0.31 实现视口块校验与句柄缓存，
v0.32 已把多级金字塔改为逐层逐行流式构建，v0.33 进一步把窗口映射、预取、校验和 LOD
合并为统一异步视口流水线。下一阶段优先增加二维空间瓦片目录和跨请求优先级合并。

## v0.30 异步视口 LOD 调度

同步 `Grid::read_overview()` 增加可选 `ProgressCallback` 与 `CancelCallback`，不改变原
C ABI。持久全幅复制、金字塔聚合和精确源节点分箱均在输出行边界检查取消；精确聚合
扫描很高的单个分箱时每 64 个源行再次检查。并行路径用原子完成行计数和串行化进度回调
报告单调 0～1 值，取消异常会停止其他工作线程并在全部 join 后向任务层传播。

公共 `dt_grid_read_overview_async()` 复制概览选项、共享持有源 `Grid`，并在任务内部
分配紧密排列的 double 结果。新的 `DT_TASK_RESULT_GRID_OVERVIEW` 与
`dt_grid_overview_view` 避免调用方输出数组跨越线程生命周期；getter 只借用任务拥有的
只读指针，`dt_task_destroy()` 是唯一释放边界。任务状态锁为结果写入与 getter 建立
happens-before，取消或失败不把未完成数组标记为可取得。

GUI 把旧同步读取从 `WM_PAINT` 中移出。每个活动请求记录源句柄、源窗口、输出尺寸、
专题类型和源元数据；完全匹配时复用，变化时请求取消并转入退休队列。队列达到上限时
暂缓提交，25 ms 定时器回收完成任务后触发一次重绘，因而快速滚轮不会无界创建线程。
消费结果前再次核对当前源和专题，旧结果绝不覆盖新视图；后台计算时继续绘制上一幅缓存。
清除/替换 GRID、切换专题和析构路径都统一请求取消，最终析构会等待并回收全部任务。

4096×4096→512×512 本机基准：同步自动并行 14.69 ms，异步提交 112 μs、后台完成
15.58 ms，异步输出校验一致；立即取消全窗口 1×1 聚合约 0.36 ms，状态为
`DT_TASK_CANCELLED`。后续磁盘瓦片版本可复用同一任务结果和 GUI 淘汰协议，只替换数据源
调度器。

## v0.31 视口块校验与句柄缓存

`Grid::verify_window()` 把二维行列窗口拆成逐行连续字节区间，再映射到 DGRIDB 既有的
4 MiB 原始数据块并去重。校验读取的是相交块完整字节，因此即使窗口只触及块的一小部分，
也能复用保存时的校验值；结果分别统计块总数、冷校验、缓存命中和实际检查字节数。
算法只建立与相交块数线性相关的小型索引，不复制窗口或完整 GRID。

每个映射 `Grid` 保存 0=未知、1=通过、2=失败的块状态，并以独立互斥量保护。锁不覆盖
磁盘哈希过程，以免一个慢块阻塞其他窗口；两个线程可能同时验证同一未知块，但最终状态
一致。通过状态可在句柄生命周期内复用，失败状态用于快速阻断后续读取。任何节点写入会
清除校验表和缓存能力，避免私有写时复制页面与源文件校验值混用。禁止校验与写入并发。

同步 C ABI 直接返回 64 字节 `dt_grid_verify_result`；异步任务共享持有源 GRID，按块报告
进度并在块边界响应取消，结果类型为 `DT_TASK_RESULT_GRID_VERIFICATION`。概览标志
`DT_GRID_OVERVIEW_VERIFY_SOURCE_BLOCKS` 在读取前完成同一源窗口校验，并把校验映射到
前 20% 进度；后续计算屏蔽该控制位，因此仍可走持久概览或金字塔快速路径。

GUI 只在 `dt_grid_info.flags` 声明块校验能力时自动设置该标志。过期请求继续遵守 v0.30
取消/淘汰协议；损坏请求不发布新缓存，保留上一幅有效位图并抑制相同键的重复失败重试。
校验只覆盖原始 double 载荷，不对持久概览和金字塔载荷作额外承诺。

8192×4096、343.19 MiB 基准中，12 个相交块（48 MiB）冷校验 58.56 ms，缓存复查约
6 μs，完整扫描 384.06 ms，冷窗口路径约快 6.56×。

## v0.32 DGRIDB 流式金字塔保存

旧 `Grid::save_binary()` 先在内存中构造所有 2× 平均金字塔层，再统一写入临时文件，
额外节点数趋近原 GRID 的三分之一。v0.32 只预计算每层尺寸、比例尺和对齐文件偏移；
第一层直接从 `GridStorage` 读取，后续层通过同一读写流从已完成的上一层读取两行，计算
一行后写入当前层。各层完成时 flush，下一层才读取，峰值工作集约为两条源行加一条输出
行，且奇数行列、NoData 排除和 long double 累加语义与旧实现一致。

头部、概览、目录、校验表和原始节点位置保持 `DGRIDB 1` 兼容。临时文件写完并 flush
后才处理当前句柄映射和原子替换：若原始节点映射目标文件，先实体化 COW 页面并释放旧
金字塔映射；替换成功后重新映射新原始节点和各层。普通内存 GRID 不映射新金字塔，避免
另一个句柄随后覆盖目标时被活动 section 阻断；文件本身仍含完整金字塔，重新加载即可用。

8192×4096 基准的旧金字塔临时数组理论值为 85.00 MiB，逐行缓冲约 0.078 MiB，保存
0.576 s；4096×2048 由 21.00 MiB 降至 0.039 MiB，保存 0.158 s。测试覆盖首次保存、
旧 v0.28 文件兼容、映射修改后同路径原子覆盖、覆盖后重映射能力、另存、金字塔数值、
校验、损坏与截断拒绝。

## v0.33 统一异步视口流水线

过去 GUI 先同步调用 `dt_grid_get_view_window()` 建立缓存键，再自行决定预取、在概览选项
中附加块校验，最后提交 `dt_grid_read_overview_async()`。功能正确，但磁盘访问策略分散在
UI 和 DLL 两层，其他集成方也必须复刻同一套顺序。v0.33 新增
`dt_grid_read_view_async()`：请求复制世界范围、输出尺寸、概览方法和执行策略，并共享持有
源 `Grid`；工作线程重新执行仿射裁剪，随后按标志预取、校验相交原始块并读取 LOD。

若启用校验，校验进度线性映射到 0～0.3，LOD 映射到 0.3～1；否则 LOD 占完整范围。
两个阶段复用同一个取消原子量。任务只有在全部阶段成功后才设置 `view_result_ready`，所以
无交集、损坏、非法尺寸和取消均不会暴露部分像素。`dt_grid_view_result` 同时携带实际
`dt_grid_window`、任务自有只读矩阵、`dt_grid_overview_result` 与可选
`dt_grid_verify_result`；getter 在任务互斥锁下发布结果，销毁任务是唯一释放边界。

GUI 仍在前台快速计算窗口用于去重缓存键，后台统一任务再次计算并在发布时核对实际窗口，
防止未来数据源或策略扩展造成键与结果漂移。原来的上一帧保留、最多三个退休任务、25 ms
轮询、协作取消和同键失败抑制均保持。该版本先统一单视口任务语义；下一阶段在这个入口
之下加入二维瓦片目录、中心优先队列和跨视口请求合并，无需再次改变 GUI 的结果生命周期。

4096×4096、1024×768 源窗口到 512×512 的本机基准中，同步局部 LOD 为 2.28 ms，统一
任务提交 54.3 μs、后台完成 2.65 ms，抽样校验和误差为 0。额外时间主要来自线程创建、
任务状态同步和结果发布；UI 线程只承担微秒级提交。

## 约束 Delaunay 后续阶段

独立 CDT 后端、折线约束、外边界、孔洞、稳定约束 ID、域内查询和 DCDT 文本已在
v0.7 完成；TIN→CDT、域内采样、CDT→GRID/等高线和 GUI 单约束编辑已在 v0.8
完成；稳定 ID 约束更新、可选新旧域影响差分和 GUI 约束顶点移动已在 v0.9 完成。
共享约束顶点引用查询、默认保护、显式单约束脱离及 GUI 单击安全删除已在 v0.10
完成。受控删除复用候选状态构建：先检查共享引用，再从目标点序列删除一个出现，
最后统一校验最小点数、零长度边、孔洞依赖和交叉拓扑，失败不会改变 generation。
v0.11 增加原子批量约束事务，按顺序组合新增、更新和删除，在一个候选状态中统一
验证并只重建一次；失败不消耗稳定 ID，成功时 generation 只增加一次。GUI 增加
12 条平行断裂线的批量演示并报告耗时，规模基准支持逐条与批量路径对比。
后续继续增加：

- 百万点约束插入、删除的局部拓扑更新与影响结果；
- 未分段交叉约束的交点高程策略；
- GDAL 约束交换与约束属性字段；
- 二进制约束数据块和流式空间索引。

对外 C ABI 保持不暴露 CGAL 类型，因此可在不影响现有调用方的情况下增加 CDT
上下文类型或创建选项。
