# 设计与后续路线

## 当前结构

```text
调用系统 / GUI
      |
稳定 C ABI + Legacy 包装
      |
Context、错误处理、结果对象
      |
CGAL Delaunay hierarchy ---- Boost face R-tree
      |                         |
顶点 ID / Z                 范围相交查询
      |
DTIN / DTMESH / XYZ 持久化
      |
GRID / Contour / Async Task（v0.3）
      |
可选 GDAL Adapter + CRS WKT（v0.4）
      |
独立 Constrained Delaunay + Domain Marking（v0.7）
```

核心是 2.5D 数据模型：CGAL 顶点位置只保存 XY，自定义顶点信息保存 Z 和稳定 ID。
面不复制 XYZ。R-tree 保存面 XY 包围盒和 CGAL 面句柄；编辑前删除旧面索引项，
编辑后只插入新增面。

v0.3 在 TIN 上方增加独立 `Grid` 和 `ContourSet` 数据对象。三者没有继承关系，
通过显式转换函数连接，避免把“等高线反推地形”误表示为无损类型转换。耗时转换
可由任务对象持有源数据共享生命周期，并通过协作取消停止。

v0.4 将 CRS 作为 TIN、GRID、ContourSet 的 UTF-8 WKT 元数据保存，转换时传播，
不参与几何计算。GDAL 适配器位于核心数据对象之外；关闭 `DT_WITH_GDAL` 时保留
相同 C ABI 导出并返回 `DT_E_UNSUPPORTED`。GeoTIFF 角点 geotransform 与本库节点
geotransform 在边界处做半像元中心换算。COG 使用 GDAL `CreateCopy`，GeoPackage
等高线使用 `LineStringZ + elevation + closed` 模式。

v0.7 增加独立 `CdtContext`，使用 CGAL
`Constrained_Delaunay_triangulation_2`。普通 TIN 保持原层次 Delaunay 与 R-tree
路径，避免约束能力改变既有动态编辑性能和 ABI。CDT 将断裂线与域边界分开：断裂线
只强制成边；外边界/孔洞边被记录为域屏障，从无限面广度优先遍历，每跨一次屏障
嵌套层级加一，奇数层为域内。

约束增删会复制基础点和约束记录，在候选 CDT 中完成插入、域标记和统计后再交换
状态，因此交叉约束或坏数据不会留下半成品。当前未分段交叉必须由调用方先加入共享
交点。DCDT 文本保存基础点、约束稳定 ID、类型、闭合标志、XYZ 和 CRS。

## 并发

每个上下文具有读写锁。当前安装的 GCC 14 MinGW 与 CGAL 6.0.1 的非平凡 TLS ABI
不兼容，因此 MinGW 构建定义 `CGAL_HAS_NO_THREADS`，并在库内用全局递归互斥锁
串行保护 CGAL 调用。MSVC/更新 CGAL 工具链验证后可恢复不同上下文的并行查询。

## 已知限制

- CDT v0.7 已支持断裂线、外边界和孔洞，但约束增删当前为全量重建，尚未局部化；
- 未分段的交叉约束当前拒绝，调用方需在交点处分段；
- GRID 当前为内存连续 `double` 数组；GDAL I/O 分行访问，但句柄本身尚未外存化；
- GRID→TIN 仍是普通 Delaunay，NoData 空洞只能拒绝或由调用方显式允许桥接；
- 当前等高线不输出完全水平平台的边界；
- `.dtin` v1 保存点集并在加载时重建，不保存逐面拓扑；`.dtmesh` 保存显式
  三角形并在打开时逐面验证，但当前仍只接受由这些顶点形成的 Delaunay 拓扑；
- 范围查询结果尚未提供流式游标；
- R-tree 当前为逐面动态索引，尚未实现压缩的块级索引；
- 旧接口使用全局默认上下文并串行执行；
- 凸包外插入的“边界”只返回可绘制的有限边，不表示无穷边。

## GUI 演示

当前已实现无额外框架依赖的 Win32/GDI 2D/3D 演示程序，验证 DLL 集成、视口范围
检索、地形分级着色、编辑影响显示、保存加载及三维漫游。二维视口结果超过45,000
个三角形时等间隔抽样；三维使用模型中心归一化、轨道相机、透视投影、画家算法
深度排序和约18,000面固定渲染预算。相机数学位于 `gui/terrain_3d.hpp`，不依赖
Win32，并由独立单元测试覆盖。

三维视图支持鼠标环视/平移/缩放、键盘水平漫游与升降、0.5～8倍垂直夸张和相机
适屏复位。二维编辑仍为主工作视图，选择查询、插入、删除或框选放大时自动返回
二维，避免把透视屏幕坐标误当作地形 XY。

v0.6 在同一 GUI 中增加 TIN、GRID、等高线三类图层状态。GRID 句柄在演示层缓存
最多400万个双精度值，生成不超过512×512的高程位图，再用 GDI `PlgBlt` 将源
矩形映射到六参数仿射变换的三个世界角点；因此旋转、错切 GRID 不会被强制画成
轴对齐矩形。等高线直接借用句柄的稳定点数组，按约20万显示顶点预算抽样，不复制
完整线集。

转换菜单调用既有稳定 C ABI：TIN→GRID 自动生成最长边401节点的规则网，等高距
按高程范围取 `1/2/5×10ⁿ` 级数。图层范围参与二维全图计算。TIN 重建或动态编辑
会销毁过期 GRID/等高线；GRID→TIN 保留源 GRID，但清除过期等高线。DGRID 与
DCONTOUR 文本可独立导入导出并与 TIN 叠加。

v0.7 GUI 增加 DCDT 打开/保存与约束图层。域内三角网使用紫色线框，外边界使用
青色粗线、孔洞使用洋红粗线、断裂线使用橙色粗线；范围查询只取域内三角形，孔洞
不会被绘制为有效地形。CDT 与普通 TIN 是可独立显隐和叠加的两个对象。

后续面向生产级千万点连续可视化时，可将 GDI 渲染层替换为 OpenGL/Direct3D，
并加入分块 LOD、后台加载、GPU 拾取和贴地碰撞：DLL 与 GUI 之间的 C ABI、视口
查询和编辑影响数据不需要改变。

## 约束 Delaunay 后续阶段

独立 CDT 后端、折线约束、外边界、孔洞、稳定约束 ID、域内查询和 DCDT 文本已在
v0.7 完成。后续继续增加：

- 百万点约束插入、删除的局部拓扑更新与影响结果；
- 约束顶点保护和受控删除规则；
- 未分段交叉约束的交点高程策略；
- CDT→GRID/等高线转换与 GDAL 约束交换；
- 二进制约束数据块和流式空间索引。

对外 C ABI 保持不暴露 CGAL 类型，因此可在不影响现有调用方的情况下增加 CDT
上下文类型或创建选项。
