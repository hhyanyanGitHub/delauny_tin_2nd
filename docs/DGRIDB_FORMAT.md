# DGRIDB 1 二进制 GRID 格式

DGRIDB 面向本机大 GRID 的快速持久化、写时复制映射和屏幕概览。它不是跨端序交换
格式；需要长期人工审阅或异构系统交换时仍可使用 DGRID 文本、GeoTIFF/COG。

## 文件布局

所有整数为无符号小端序，浮点数为 IEEE-754 little-endian `double`。文件由三段组成：

1. 固定 65,536 字节头；
2. 最多 512×512 个 `double` 的全幅平均概览，随后补零到 65,536 字节边界；
3. `width * height` 个按行优先排列的原始 `double` 节点。

| 偏移 | 大小 | 内容 |
|---:|---:|---|
| 0 | 8 | `DGRIDB1\0` |
| 8 | 4 | 格式版本，当前为 1 |
| 12 | 4 | 文件头大小，固定 65,536 |
| 16 | 4 | 端序标记 `0x01020304` |
| 20 | 4 | GRID 语义标志；文件中只允许 `DT_GRID_HAS_NODATA` |
| 24 | 8 | width |
| 32 | 8 | height |
| 40 | 48 | 六参数节点仿射变换 |
| 88 | 8 | NoData |
| 96 | 8 | UTF-8 CRS WKT 字节数，不含 NUL |
| 104 | 8 | 概览宽度，`min(width,512)` |
| 112 | 8 | 概览高度，`min(height,512)` |
| 120 | 8 | 概览偏移，当前固定 65,536 |
| 128 | 8 | 原始节点偏移，按 65,536 字节对齐 |
| 136 | 8 | 原始节点总数 |
| 144 | 8 | 有效节点数 |
| 152 | 8 | NoData 节点数 |
| 160 | 8 | 有效节点最小值 |
| 168 | 8 | 有效节点最大值 |
| 176 | 8 | 有效节点平均值 |

加载器校验魔数、版本、端序、头校验、尺寸乘法、段偏移、统计计数和实际文件大小。
头校验不覆盖概览及原始节点；需要传输级完整性时应在文件外增加 SHA-256 等校验。

## 映射与编辑语义

Windows 加载器以 `PAGE_WRITECOPY` / `FILE_MAP_COPY` 映射原始节点。读取只在访问相应
页面时由系统调页；`dt_grid_write_window()` 修改私有页，不会改变源 DGRIDB。所有现有
GRID 查询、转换和分析函数继续接收同一个 `dt_grid_handle`。

`dt_grid_info.flags` 的 `DT_GRID_STORAGE_MEMORY_MAPPED` 表示当前节点仍由映射支持，
`DT_GRID_HAS_PERSISTENT_OVERVIEW` 表示可直接复用内置全幅平均概览。这两个标志只用于
输出，不应传给 `dt_grid_create()`。

显式调用 `dt_grid_save_binary()` 才提交数据。保存先在目标目录完整写临时文件并刷新，
成功后原子替换目标；失败时已有目标不变。若保存目标就是当前映射源，库在临时文件
完成后将当前私有视图实体化并解除映射，再执行替换，因此该特殊路径会短时需要一份
完整节点内存。另存为其他文件不需要解除源映射。

## 持久概览

保存时用 `DT_GRID_OVERVIEW_AVERAGE` 为完整源范围生成最多 512×512 的概览，并记录
完整源统计。加载后，以相同尺寸、平均方法、默认 NoData 策略请求完整源范围时，
`dt_grid_read_overview()` 直接复制该概览；局部视口或其他方法仍读取原始映射节点。
任意节点写入会使旧概览失效；下一次二进制保存会重新生成并嵌入新概览。
