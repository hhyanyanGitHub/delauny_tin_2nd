# DGTILE 1 稀疏显示瓦片包

`DGTILE 1` 是 GRID 交互显示的可重建旁车缓存，不是地形数据交换格式。正式高程、仿射、
CRS 和 NoData 仍以 DGRIDB、DGRID、GeoTIFF 等源数据为准。关闭程序后可以安全删除
`.dgtile`；下次浏览时库会从源 GRID 重新生成需要的瓦片。

## 设计目标

- 只保存用户实际浏览过的空间瓦片，不预先生成整幅金字塔；
- 打开时只扫描小型记录头，瓦片像素在命中时才读取；
- 负载损坏、源数据更新、容量耗尽或旁车不可写不影响正式 GRID；
- 保持 v0.34 的中心优先、在途合并、内存 LRU 和任务结果生命周期。

所有整数和 `double` 均按小端序写入。当前实现面向 Windows x64；不同端序的平台必须先
转换，不能直接映射解释。未声明的保留字节写零，读取方不得依赖其内容。

## 文件头

文件头固定 4096 字节，FNV-1a 64 位校验覆盖完整文件头；计算时先把校验字段置零。

| 偏移 | 长度 | 内容 |
|---:|---:|---|
| 0 | 8 | `DGTILE1\0` |
| 8 | 4 | 版本号 `1` |
| 12 | 4 | 文件头长度 `4096` |
| 16 | 4 | 端序标记 `0x01020304` |
| 20 | 4 | 保留 |
| 24 | 4 | 瓦片输出宽度 |
| 28 | 4 | 瓦片输出高度 |
| 32 | 8 | 源 GRID 宽度 |
| 40 | 8 | 源 GRID 高度 |
| 48 | 56 | 保留 |
| 104 | 8 | 源指纹 |
| 112 | 8 | 已提交记录数 |
| 120 | 8 | 已提交文件字节数 |
| 128 | 8 | 文件头 FNV-1a 64 校验 |
| 136 | 3960 | 保留 |

源指纹包含应用提供的 `source_revision`、行列数、NoData 语义、六参数仿射、NoData 值、
CRS 字节和最多 64×64 个确定性源样本。抽样不是密码学内容哈希，也不保证发现任意未抽样
节点的修改；管理内存 GRID 时，调用方应在数据发生语义变化后更新 `source_revision`。

## 瓦片记录

文件头后是连续的“128 字节记录头 + 行优先 double 像素负载”。记录不压缩；打开包时只
读取记录头并建立内存索引，后出现的同键记录覆盖先前索引。记录头校验同样采用 FNV-1a
64，计算时先把偏移 120 的字段置零；像素负载另有 FNV-1a 64 校验。

| 偏移 | 长度 | 内容 |
|---:|---:|---|
| 0 | 8 | `DGTREC1\0` |
| 8 | 4 | 记录头长度 `128` |
| 12 | 4 | 保留 |
| 16 | 8 | 2 的幂次 LOD 比例 `scale` |
| 24 | 8 | 瓦片 X |
| 32 | 8 | 瓦片 Y |
| 40 | 4 | 概览方法 |
| 44 | 4 | 影响数值的概览标志 |
| 48 | 8 | 实际瓦片宽度 |
| 56 | 8 | 实际瓦片高度 |
| 64 | 4 | 概览结果标志 |
| 68 | 4 | 保留 |
| 72 | 8 | 有效像素数 |
| 80 | 8 | NoData 像素数 |
| 88 | 8 | 最小值 |
| 96 | 8 | 最大值 |
| 104 | 8 | 均值 |
| 112 | 8 | 像素负载 FNV-1a 64 校验 |

## 提交、恢复与容量

写入先把记录头和像素追加到旧的“已提交文件字节数”，最后更新并刷新 4096 字节文件头。
因此未被文件头提交的尾部字节会在下次追加时覆盖。延迟发现的坏负载在启用
`RESET_CORRUPTED` 时从索引删除，重算后追加新记录。

`maximum_file_bytes` 默认为 2 GiB，允许范围是能容纳头和一个记录头的最小值至 1 TiB。
现有文件超过新配置的上限时仍可只读命中，但停止追加；新记录放不下时增加跳过计数。
`dt_grid_view_cache_compact()` 会按稳定键序把内存索引中的最后版本写入同目录临时文件，
逐块复核负载校验，刷新后原子替换原包。历史同键记录被回收；当前索引指向的坏负载被
省略并在后续请求中重建。结果报告压缩前后字节数和各类丢弃数。彻底清理仍可关闭句柄后
删除旁车文件。

## 并发约束

同一缓存句柄的瓦片读取持有共享文件锁，追加和压缩持有独占文件锁；压缩后加载会按键
重新取得最新偏移，不使用替换前的陈旧描述。单进程内，同一规范化路径最多一个可写包。
Windows 下可写句柄还持有由规范化绝对路径哈希得到的命名互斥量，另一进程并发写入会在
创建时返回 `DT_E_IO`。只读包不获取写者互斥量；若应用需要在写入同时跨进程读取，应自行
规定文件快照时机，最简单的方式是在写者关闭后再打开只读句柄。非 Windows 构建当前仅
保证进程内单写。

## 与 DGRIDB 的关系

GUI 为 `terrain.dgridb` 自动使用 `terrain.dgridb.dgtile`。二者没有嵌套或事务依赖：
DGRIDB 保存完整数据和内置平均金字塔，DGTILE 保存按视口、方法和显示 LOD 切出的热点
瓦片。DGRIDB 被替换后，文件大小/修改时间形成的 GUI revision 与源指纹会使旧旁车失效
并自动重建。复制数据集时可以复制旁车以保留热点，也可以只复制 DGRIDB。
