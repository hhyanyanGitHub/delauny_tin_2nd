# Windows GUI 演示说明

可执行文件：`build/dterrain_demo.exe`。

## 操作

| 控件 | 功能 |
|---|---|
| 模拟10万 | 生成约10万点的起伏地形并建网 |
| 模拟100万 | 生成约100万点，用于大数据交互演示 |
| 清空 | 清除当前三角网 |
| 查询模式 | 单击显示最近顶点和覆盖三角形 |
| 插入模式 | 单击插入一个按模拟地形函数计算高程的新点 |
| 删除模式 | 单击删除 XY 距离最近的顶点 |
| 框选放大 | 按住鼠标左键拖出矩形，松开后将选区放大并按画布比例适屏 |
| 全图 | 恢复到数据完整范围 |
| 导入XYZ | 导入 `.xyz/.txt/.csv` 散点并立即自动构网 |
| 保存网格 | 保存 `.dtmesh/.txt` 文本三角网或 `.dtin` 二进制网格 |
| 打开网格 | 打开并校验 `.dtmesh/.txt`，或加载 `.dtin` |
| 切换3D/切换2D | 在二维编辑视图与透视三维地形视图之间切换 |
| 高程×n | 依次选择 0.5、1.0、1.5、2.0、3.0、5.0、8.0 倍垂直夸张 |

“分析”菜单提供任意剖面、多边形面积/土方量测、TIN/CDT 精确范围裁剪、双 GRID
平移配准与自适应误差评估；剖面和量测可导出 CSV，并会自动回到二维视图。

鼠标滚轮以指针位置为中心缩放。右键或中键拖动平移。在“框选放大”模式下，
按住左键拖出青色虚线选择框，松开后自动放大；选择过程中按 `Esc` 可取消。

XYZ 文件每个有效行包含 `x y z`，也接受逗号、分号和制表符；空行和以 `#`
开头的行会被忽略。文本三角网格式见 [TEXT_FORMATS.md](TEXT_FORMATS.md)。
便携目录附带的 `sample_data/sample_points.xyz` 可直接用于导入测试。

## 地形转换与图层

“地形转换”菜单提供以下自动化工作流：

| 命令 | 行为 |
|---|---|
| TIN → GRID | 按 TIN 范围生成最长边 401 节点、保持 XY 比例的规则 GRID |
| GRID → TIN | 以有效 GRID 节点重建普通 TIN；NoData 不自动形成硬孔洞，真实孔洞请使用独立 DCDT 约束网 |
| 从 TIN 生成等高线 | 扫描 TIN 高程范围并自动选取易读等高距 |
| 从 GRID 生成等高线 | 使用 GRID 有效高程范围自动选取等高距 |
| 等高线 → TIN | 以等高线折点和线高程重建普通 Delaunay；保留源等高线用于叠加检查 |
| 等高线 → GRID | 先由等高线折点建立临时 TIN，再生成最长边 401 节点 GRID |
| 从当前 TIN 创建约束网 | 复制普通 TIN 的全部顶点和 CRS，替换现有 CDT 基础点并清除旧约束 |
| 约束网 → GRID | 只插值有效域；外边界以外和孔洞内部写为 NoData |
| 从约束网生成等高线 | 只扫描有效域，等高线不会跨越外边界或孔洞 |

自动等高距使用 `1/2/5 × 10ⁿ` 级数，目标约 12 个高程级别。“图层”菜单可分别
显示或隐藏 TIN、GRID 高程着色和等高线；状态栏用 `T/G/C` 表示当前可见图层。
“全图”会计算所有可见二维图层的联合范围。

TIN 被重新生成、导入、打开或动态编辑后，已有 GRID 和等高线属于过期派生结果，
程序会自动释放这些图层。GRID→TIN 会保留源 GRID，清除不再匹配的等高线。
等高线→TIN 会保留作为输入的等高线、释放旧 GRID 和 CDT；等高线→GRID 保留源
等高线及现有 TIN，便于对照显示。

等高线反向转换属于近似重建：GUI 使用原折点，不会恢复等高线之间已经丢失的极值，
也不会强制等高线段成为普通 TIN 边。若输入只有一条直线或全部折点共线，DLL 会
拒绝转换并保留原数据。

## GRID、等高线与 GDAL 格式交换

“数据交换”菜单可独立导入和导出 `DGRIDB 1` 二进制映射 GRID、`DGRID 1` 文本和
`DCONTOUR 1` 文本。导入 GRID 或
等高线不会强制删除现有 TIN，因此可以叠加检查。详细格式见
[TEXT_FORMATS.md](TEXT_FORMATS.md) 与 [DGRIDB_FORMAT.md](DGRIDB_FORMAT.md)。
安装目录的 `sample_data/sample_grid.dgrid` 和 `sample_contours.dcontour` 可直接
用于验证两个菜单入口。

“数据交换”菜单还提供“打开约束网 DCDT / DCDTB”和“保存约束网 DCDT / DCDTB”。
程序按扩展名选择可读文本或带范围目录、全载荷校验的二进制格式，保存对话框默认
`.dcdtb`。打开后不替换
普通 TIN，可通过“图层→约束 Delaunay”独立显隐或叠加比较。安装目录的
`sample_data/sample_constraints.dcdt` 包含外边界、孔洞和断裂线，可直接验证。

使用 `DT_WITH_GDAL=ON` 构建时，“数据交换”菜单还提供：

| 命令 | 数据与行为 |
|---|---|
| 导入 GDAL 栅格 | 打开 GeoTIFF、DEM、IMG、ASCII Grid、VRT 等，读取第 1 波段为 GRID |
| 导出 GeoTIFF（DEFLATE） | 保存当前 GRID，使用 TILED、DEFLATE、BIGTIFF=IF_SAFER |
| 导出 Cloud Optimized GeoTIFF | 保存当前 GRID，使用 COG、DEFLATE、BIGTIFF=IF_SAFER |
| 导入 GDAL 线矢量等高线 | 打开 GPKG、SHP、GeoJSON、KML 等，读取 `elevation` 和 LineStringZ |
| 导出 GeoPackage 等高线 | 写入 `contours` 图层、`elevation/closed` 字段并建立空间索引 |
| 当前 TIN/GRID/等高线重投影到 EPSG | 输入整数 EPSG，生成目标坐标系对象并替换同类图层 |

程序启动时检查 GDAL 以及 `GTiff`、`COG` 和 `GPKG` 写驱动。普通构建或缺少相应驱动时，
对应菜单项按能力置灰，DGRID、DCONTOUR、DCDT 和 DCDTB 操作仍可使用。GDAL 格式
导入不会删除普通 TIN、CDT 或另一种派生图层，因此状态栏可继续显示 `T/G/C/D`
组合并进行叠加对照。导入失败时原图层保持不变。

GDAL 栅格像元中心对应本库 GRID 节点，读写时自动处理半像元偏移。GUI 使用第 1 波段。
重投影必须从菜单显式发起，源图层必须已有有效 CRS；GRID 默认采用双线性和 GDAL 自动
推导的北向上目标网格，TIN 在目标平面重新构建 Delaunay，等高线保持折线分组。成功后
其他地形图层自动隐藏，避免把不同坐标系误叠加。工程级节点对齐应调用 DLL 的显式目标
GRID 接口，而不是依赖 GUI 自动网格。

## 大 GRID 的自动 LOD 预览

v0.26 起，导入或生成 GRID 后，GUI 不再把整幅 double 数组复制到显示层。v0.27 进一步
在每次滚轮缩放、框选放大、全图复位或平移结束后调用 `dt_grid_get_view_window()`，把当前
世界视口精确裁到 GRID 源行列窗口，再由 `dt_grid_read_overview()` 聚合为最多 512×512
的小型缓存。位图按该局部源窗口的六参数仿射角点绘制；超过 2000 万节点的 DGRID、
GeoTIFF/COG、高差 GRID 和地形专题既能全图预览，也能在放大后逐级获得局部细节。

| 图层 | LOD 方法 | 原因 |
|---|---|---|
| 高程 GRID | 有效值平均 | 保留连续地形的区域趋势，并取得完整源范围高程统计 |
| 坡度、阴影、高差 | 有效值平均 | 降低抽样闪烁，表达分箱内平均分析量 |
| 坡向 | 最近邻 | 坡向是环形角度，避免 359° 与 1° 被算术平均为 180° |

平均概览的每个源节点恰好进入一个输出分箱，NoData 默认被忽略；完全无有效值的分箱
显示深色背景。状态栏以“局部LOD 源窗口→预览尺寸”报告当前层级；平移拖动期间复用
旧缓存，松开后刷新一次。自动等高距使用完整源概览
调用同时返回的完整源窗口最小/最大高程，因此“从 GRID 生成等高线”不再受旧的 GUI
整幅缓存上限限制。LOD 只用于二维显示，不会修改或降低 DLL 中原 GRID 的分辨率。

旋转或剪切 GRID 的世界视口在逆仿射后通常不是矩形。DLL 会把该四边形裁到源节点域，
而不是简单读取整幅或假定北向上；完全移出 GRID 时不读取高程。局部缓存额外扩展两个
源节点，减少边缘取整产生的空缝。若视口源窗口小于 512×512，则按真实节点数显示，
不会为连续高程做无意义上采样。

v0.30 起视口 LOD 在后台任务中生成。滚轮、拉框、窗口大小变化或平移结束只提交新请求，
画布继续显示上一幅有效缓存，状态栏显示“异步LOD xx%”；新结果完成后再一次性替换。
若视口在任务完成前再次变化，GUI 会协作取消旧任务并只接受当前 GRID、源窗口、输出尺寸
和专题类型完全匹配的结果。待回收任务数量受限，25 ms 定时轮询不会阻塞绘制消息。切换
专题、替换/清除 GRID 或关闭窗口时同样会取消并安全回收后台 LOD，不发布部分位图。

v0.31 起，带 `DT_GRID_HAS_BLOCK_CHECKSUMS` 能力的 DGRIDB 在每个异步 LOD 请求开始时自动
校验当前源窗口相交的 4 MiB 原始节点块。首次浏览某区域会读取并校验冷块，同一已加载
句柄再次浏览时直接命中线程安全缓存；状态栏能力说明为“按需块校验缓存”。若某块损坏，
当前 LOD 不会发布，画布保留上一幅有效缓存，并抑制同一源窗口的无休止重复提交。切换到
其他窗口仍可继续验证和显示未受影响区域。

v0.29 的 `.dgridb` 导入在 Windows 下把原始节点和多级概览写时复制映射到
`dt_grid_handle`，并直接使用文件内最多 512×512 的全幅平均概览完成首次适屏。状态栏
追加“写时复制映射”“内置概览”“多级金字塔”和“块校验”；局部缩放会选择仍覆盖
屏幕输出尺寸的最低分辨率层，并显示“金字塔LOD 源窗口→预览尺寸”。分析、转换与导出
仍读取完整原始 double 节点，不把显示金字塔当作分析数据。对映射 GRID
执行窗口编辑只改变私有页面；通过“导出 GRID”显式保存后才产生新 DGRIDB。若覆盖
当前源文件，库会在临时文件写完后解除映射并原子替换，期间可能短时实体化完整节点；
大数据更建议先另存为新文件。

v0.32 起，“导出 GRID”生成 DGRIDB 金字塔时按层、按行直接写临时文件，不再额外分配
约三分之一原始节点量的完整金字塔数组。8192×4096 基准的金字塔临时内存理论值由
85.00 MiB 降至约 0.078 MiB，文件格式和再次打开后的 LOD 行为不变。普通内存 GRID
导出后若要立即使用文件内金字塔，应重新打开刚保存的 `.dgridb`；覆盖当前映射文件时
句柄会自动重挂新文件，无需手工重开。

v0.33 起，缩放、拉框、适屏、窗口大小变化和平移结束后的后台刷新统一调用
`dt_grid_read_view_async()`。DLL 在同一个任务内重新确认世界视口对应的源窗口，按请求
执行映射页预取和 DGRIDB 相交块校验，再生成金字塔或精确 LOD；GUI 一次取得实际窗口、
像素、统计和校验摘要。画布仍持续显示上一幅有效缓存，连续操作会取消过期任务，损坏或
失败请求不会覆盖画面。对用户而言操作手势不变，但磁盘访问、校验与 LOD 的先后关系已
由 DLL 统一管理，状态栏进度覆盖完整流水线。

“数据交换→验证 DGRIDB 数据块”会完整扫描文件内每个 4 MiB 原始节点块并与保存时的
校验表比对。普通打开不会自动做这一步，以保持毫秒级加载；归档、复制传输后核验或怀疑
磁盘损坏时再主动执行。自动视口校验只覆盖用户实际访问的原始节点块，并不校验持久概览
或金字塔载荷，因此不是完整审计的替代品。无块校验扩展的 v0.28 文件仍可打开，但该命令
会报告不支持。8192×4096 本机基准中，12 个窗口块冷校验约 58.56 ms，缓存复查约 6 μs，
完整扫描约 384.06 ms；数值随磁盘缓存和硬件变化。

## 任意地形剖面分析

选择“分析→任意剖面（两次单击）”，在二维画布中先后单击 A、B 两点。程序沿
A—B 生成 401 个等距样本，绘制青色定位线和端点，并在画布底部显示高程曲线。
下一次单击会开始一条新剖面；第一次单击后按 `Esc` 可取消，菜单“清除剖面”可
清除已完成结果。

整条剖面固定使用一个数据源，选择优先级如下：

1. 当前可见且存在的 CDT 有效域；
2. 当前可见且存在的普通 TIN；
3. 当前可见且存在的 GRID；
4. 若以上均不可用，再按 CDT、TIN、GRID 顺序选择隐藏但存在的数据。

CDT 外边界以外或孔洞内、TIN 凸包外以及 GRID NoData 样本不会被其他图层补齐，
而是作为断点显示。这保证一条曲线具有一致的数据来源。状态栏和曲线面板显示平距、
有效样本数、最低/最高高程、净高差、累计上升、累计下降和相邻有效样本最大绝对
坡度。坡度以 `ΔZ / Δ平距 × 100%` 计算，断点两侧不跨段累计。

“分析→导出剖面 CSV”输出 UTF-8 文本，首行为数据源注释，字段包括
`index,distance,x,y,z,valid,grade_percent`。无效样本的 Z 和坡度留空；连续有效段
首点的坡度也留空。GRID 使用完整六参数仿射反算和双线性插值，并只读取每个样本
邻近的小窗口，适用于无法整体缓存的大栅格。

若剖面使用的源图层被重建、动态编辑、导入替换、转换替换或清除，程序会自动清除
旧剖面，避免把过期曲线误认为当前地形结果。

## 多边形面积与土方量测

选择“分析→面积/土方量测（逐点）”，在弹出的窗口中输入水平基准高程，然后在
二维画布中依次单击简单多边形的边界点。至少输入 3 点后按 `Enter` 闭合并计算；
`Backspace` 撤回草图末点，`Esc` 或“清除面积/土方量测”清除结果。自相交、重复边
或面积退化的边界会被拒绝。完成结果后再次单击会开始一块新的量测区域。

整块量测固定使用首点确定的数据源，优先级与剖面相同：可见 CDT、可见 TIN、
可见 GRID，其后回退到存在但隐藏的 CDT、TIN、GRID。CDT 外域/孔洞、TIN 凸包外
和 GRID NoData 不会从其他图层补齐。边界可跨越这些区域，结果会单独报告有效平面
面积和覆盖率。

结果含义如下：

| 字段 | 含义 |
|---|---|
| 多边形平面面积、周长 | 由输入 XY 边界直接计算 |
| 有效平面面积、覆盖率 | 当前数据源中可参与积分的区域及其占多边形面积比例 |
| 地表面积 | 对有效区域三维微三角形面积求和 |
| 最低/最高/平均高程 | 有效采样高程；平均值按平面面积加权 |
| 挖方 | 现状地形高于水平基准面的体积 |
| 填方 | 现状地形低于水平基准面的体积 |
| 净挖方 | 挖方减去填方，负值表示净填方 |

可用“分析→设置量测基准高程…”修改基准面并立即重算。“导出面积/土方 CSV…”输出
UTF-8 文本，前部为数据源、基准高程、各项统计和积分单元数，随后为
`index,x,y,z,valid` 边界顶点表。边界顶点无有效高程时 Z 留空；这不妨碍程序对
边界内部有效部分积分。

平面面积和周长是边界几何量；地表面积、高程与挖填方使用约 20,000 个微三角形
积分，是数值估算。孔洞/NoData 边缘按微三角形中心分类，误差受区域尺度、地形
起伏和数据分辨率影响。演示结果适合研究和方案比较，不应直接作为工程结算量。

若量测所用源图层被重建、动态编辑、导入替换、转换替换或清除，程序会自动清除
对应结果；修改无关图层不会使它失效。

## 按量测多边形裁剪或掩膜 GRID

完成一个面积/土方量测多边形后，可立即把同一 XY 边界用于当前高程 GRID：

1. “按量测多边形掩膜当前 GRID（保持范围）”保留原行列数和仿射，只把区域外节点
   写为 NoData；
2. “按量测多边形裁剪当前 GRID（紧凑适屏）”进一步缩小到多边形的源节点包络，
   更新仿射原点并自动适屏，适合局部导出；
3. “按量测多边形反向掩膜当前 GRID”保留区域外节点，可用于挖除或屏蔽施工范围。

任务异步执行并显示百分比，按 `Esc` 可协作取消；取消或失败时原 GRID、量测多边形
和其他结果保持不变。成功后当前 GRID 被原子替换，原设计面、土方差值、专题和等高线
等依赖层被清除，已消费的量测多边形也被清除。新 GRID 可继续保存 DGRID、导出
GeoTIFF/COG、生成等高线或作地形专题分析。

裁剪按 GRID 节点判定，边界节点保留，源 NoData 不会被填补。多边形在世界 XY 中
输入，程序内部使用完整六参数逆仿射，因此旋转或剪切 GRID 同样适用。若需要按工程
边界精确裁剪栅格单元面积，而非筛选节点，仍应在土方或矢量叠加流程中另作单元级
解析裁剪；v0.85 新增的精确接口针对 TIN/CDT 三角面。

## TIN/CDT 精确裁剪与双 GRID 配准

完成量测多边形后，若固定数据源是 TIN 或 CDT，可选择“精确裁剪量测范围
（TIN / CDT）”。DLL 先用范围索引筛选候选三角面，再逐面计算与量测边界的解析交集；
状态栏显示片段数、环数和精确投影面积。该命令用于核对数值与接口效果，不替换源图层。
DLL API 还支持外环加多个孔洞；GUI 当前量测工具输入单外环。

准备当前 GRID 和设计 GRID 后，可选择“配准设计 GRID 到当前 GRID”。GUI 使用 4096
个确定性样本，在自动搜索半径内估计 XY 平移及常量 Z 偏差，随后把设计面双线性重采样
到当前 GRID 的完整仿射几何并替换设计面。状态栏报告 ΔX/ΔY/ΔZ、配准前后 RMSE 和
有效样本数。配准只处理平移，不估计旋转、尺度或弹性变形；CRS 不一致会拒绝。

“自适应评估双 GRID 误差”比较当前面与设计面，至少使用 2048 个共同有效样本，最多
65536 个；RMSE 标准误差达到 0.001 当前高程单位时提前停止。状态栏报告 RMSE±标准误、
平均/最大绝对误差及有效/尝试样本数。强空间自相关、系统误差或工程验收场景仍需采用
项目规定的控制点和统计方法。

## 双表面 GRID 挖填方

双表面分析与前述“水平基准面多边形量测”不同：它直接比较两幅完整 GRID，不使用
微三角形采样预算。当前高程 `GRID` 是现状面，设计面通过“分析”菜单准备：

1. “加载设计面 DGRID…”读取规则网格文本；GDAL 构建也可选择“加载设计面
   GeoTIFF/COG…”；
2. 没有设计数据时，可选择“从现状 GRID 创建偏移设计面…”，输入设计面相对现状
   的统一高程偏移；正值表示设计面填高；
3. CRS 不一致会立即拒绝，因为 GUI 不执行隐式重投影；同 CRS 但行列数或仿射不一致
   的设计面会被保留，并提示选择“双线性对齐设计面到现状 GRID”或“最近邻对齐…”；
4. 对齐任务显示百分比，按 `Esc` 可安全取消且原设计面保持不变。双线性适合连续
   高程，最近邻适合分类/离散值；GUI 使用严格四节点 NoData 语义；
5. 对齐成功后选择“计算现状面—设计面挖填方”，观察百分比；按 `Esc` 可安全取消；
6. 完成后蓝色表示设计高于现状的填方，红色表示现状高于设计的挖方，白色接近
   零高差。状态栏显示挖、填、净方、有效覆盖率和耗时；
7. “导出双表面土方摘要 CSV…”保存面积、覆盖率、体积和高差统计；当前高差 GRID
   还可沿用专题 DGRID/GeoTIFF 导出命令；
8. “清除设计面与双表面结果”释放设计 GRID 和高差结果，恢复现状高程显示。

双表面计算默认对 NoData 使用整单元拒绝：四个角点任一现状/设计值无效，就跳过
该单元的两个三角形。覆盖率小于 100% 时应先核对两面数据范围、NoData 与工程边界。
挖填符号约定为“现状减设计”：正值是挖方，负值是填方，净方为挖减填。GUI 使用
“设置专题分析与性能参数”中的 z-factor、线程数和块高；两个面使用同一 z-factor。

## 坡度、坡向与地形法向

选择“分析→坡度/坡向分析（单击）”，程序自动回到二维视图。在地形上单击一次即
完成局部分析；之后每次单击会用新位置替换上次结果。按 `Esc` 或选择“清除坡度/
坡向结果”可清除覆盖层。

单击时按以下顺序固定选择一个数据源：可见 CDT、可见 TIN、可见 GRID，若均不可用
再按 CDT、TIN、GRID 回退到已存在但隐藏的数据。CDT 外域/孔洞、TIN 凸包外和
GRID 范围外/NoData 不会用其他图层在同一次调用中补值；当前选定源在查询点无效
时本次分析失败。可通过图层显隐明确选择叠加地形中的目标表面，结果面板会显示
最终数据源。

显示元素：

- 黄色圆点：实际查询 XY 和插值高程的平面位置；
- 青色虚线：TIN/CDT 的支撑三角形，或 GRID 的 2×2 支撑单元；
- 青色箭头：最大下降方向，箭头长度为屏幕显示长度，不代表实际坡长；
- 右上深色面板：数据源、Z、坡度角、坡向及八方位、`dz/dx`、`dz/dy` 和向上单位法向。

坡度相对水平面计量，水平面为 0°。坡向以地图 +Y 为北，顺时针表示下坡方向；
水平面显示“无唯一坡向”。TIN/CDT 在三角面内坡度恒定，查询落在边或顶点时采用
一个有效邻面；GRID 使用局部双线性导数，因此同一网格单元内数值可能随位置变化。
若当前还有完成的面积/土方面板，坡度面板自动下移，避免遮挡。

依赖的源图层发生动态编辑、重建、导入替换、转换替换或清除时，旧坡度结果自动
失效；无关图层变化不会清除它。

## 全幅地形专题图

“分析”菜单提供“生成全幅坡度专题图”“生成全幅坡向专题图”和“生成阴影地形图
（当前光照参数）”。已有 GRID 时直接分析；只有普通 TIN 或有效 CDT 时，程序先自动
按当前范围生成最长边 401 节点的高程 GRID，再计算专题结果。若两种三角网同时
存在，优先使用有效 CDT。

“设置专题分析与性能参数”依次设置高程单位系数 z-factor、太阳方位角、太阳高度角、
工作线程数和分块行数。
z-factor 默认 1.0，必须大于 0；例如 XY 为米而 Z 为毫米时应填 0.001。方位角以
北为 0°顺时针，任意输入会归一化到 0～360°；高度角范围为 -90～90°，默认光照
仍为 315°/45°。线程数填 0 自动、1 单线程或 2～64；块高填 0 使用默认 64 行，
也可填 1～1048576。通常保留自动设置；调试复现或宿主限制 CPU 时再显式调整。

三类专题图均保持原 GRID 的尺寸、旋转/错切仿射变换、CRS 和 NoData 区域：

| 专题图 | 显示与数值 |
|---|---|
| 坡度 | 绿色到紫色固定色带，0～60°，更陡坡度使用末端颜色 |
| 坡向 | 环形色相，按北→东→南→西→北覆盖 0～360°；水平面/无效区为 NoData |
| 阴影地形 | 0～255 灰度，默认西北 315°、高度角 45° 光照 |

右上角图例给出当前量纲和固定范围，因此切换不同数据集时颜色仍可比较。“恢复显示
高程 GRID”销毁当前专题结果并恢复蓝—绿—黄—棕—白高程色带；原高程 GRID 从未
被专题结果覆盖，剖面、量测、局部坡度查询和三维视图仍以它为数据源。

“导出当前专题图 DGRID”可在所有构建中使用；启用 GDAL/GTiff 后还可导出分块、
DEFLATE 压缩的 GeoTIFF。导出的数据是实际坡度角、坡向角或 0～255 阴影值，不是
屏幕 RGB 截图，可由 GIS 软件继续统计、分级或制图。

专题计算在后台任务中运行，状态栏持续显示百分比，窗口仍可重绘和移动。计算期间
按 `Esc` 或选择“取消正在进行的专题计算”可协作取消；此时不替换当前专题结果。
若计算期间关闭窗口，程序会先取消并等待工作线程安全退出，再完成关闭。普通菜单
命令在任务结束前会被忽略，避免源 GRID 被并发替换。

分块并行不会改变输出数值或额外复制一份全幅缓冲；工作线程直接填充结果 GRID 的
不同输出行。状态栏在完成后显示本次请求的线程模式与块高。小 GRID 或块数不足时，
DLL 会自动少用线程；过小块高可能增加调度开销，过大则降低负载均衡和取消响应。

## 约束 Delaunay 图层

DCDT 图层只绘制外边界以内、孔洞以外的有效域三角形。颜色含义：

- 紫色细线：域内约束 Delaunay 三角形；
- 青色粗线：地形外边界；
- 洋红粗线：孔洞边界；
- 橙色粗线：不改变域内外的断裂线。

状态栏图层码增加 `D`。全图范围会包含可见 CDT 的全部顶点。普通查询、插入和
删除按钮仍操作普通 TIN，不会隐式修改独立 CDT。

## 交互编辑约束

“约束编辑”菜单提供“绘制断裂线”“绘制外边界”“绘制孔洞边界”“批量添加 12 条
示例断裂线”“移动约束顶点”和“选择删除约束”。
建议先执行“地形转换→从当前 TIN 创建约束网”，或打开已有 DCDT。绘制操作：

1. 选择约束类型后，在二维画布中逐点单击；黄色草图显示当前输入；
2. 按 `Enter` 或选择“完成当前约束”提交；断裂线至少 2 点，边界至少 3 点；
3. 按 `Backspace` 撤销最后一点，按 `Esc` 或菜单命令取消整条草图；
4. 选择“选择删除约束”后，在青、洋红或橙色约束线 14 像素范围内单击；

GUI 的 CDT 默认启用交叉自动节点化，交点两条线的插值 Z 容差为当前高程单位
`0.001`。高程一致的 XY 交点会自动加入相关折线；高差超限或共线重叠时整次编辑回滚。
无交叉普通断裂线采用局部 CGAL 拓扑插入；边界、改形和删除仍通过原子候选网完成。
5. 删除外边界前必须先删除依赖它的孔洞，否则 DLL 会拒绝操作并保持原网不变。

移动已有折点时，选择“移动约束顶点（两次单击）”：

1. 所有约束折点显示为白色小圆点；先在 14 像素范围内单击目标折点；
2. 被选折点显示为黄色大圆点，再单击新位置提交移动；
3. 按 `Esc` 或“取消当前约束”可取消选择；
4. 成功后约束 ID 保持不变，红色显示旧域面、黄色显示影响边界、绿色显示新域面和边；
5. 若新位置造成零长度边、未分段交叉或无效孔洞，提交被拒绝，原约束和拓扑保持不变。

安全删除单个折点时，选择“删除约束顶点（单击）”：

1. 白色小圆点表示可拾取的约束顶点，在 14 像素范围内单击目标点；
2. 普通顶点直接提交删除，并显示红/黄/绿影响结果；
3. 若该 XY 被多条约束共享，程序会显示引用数量并请求确认；选择“是”只从当前
   约束脱离，其他约束保持不变，选择“否”不改变数据；
4. 若该 XY 同时是基础地形点，删除的只是约束引用，基础点仍保留；
5. 删除会导致折线少于 2 点、闭合边界少于 3 点或产生非法拓扑时，DLL 拒绝操作并
   原子保留原网。整条约束删除仍使用“选择删除约束”。

草图点 Z 优先从当前 CDT 有效面插值，其次从普通 TIN 插值；两者均不可用时才使用
演示地形函数。约束提交或删除会释放已派生的 GRID/等高线，避免显示过期结果。
当前 DLL 对单条约束增删改仍执行候选 CDT 全量重建；移动/删除顶点为显示影响还会执行
完整的新旧域三角形差分。v0.11 的批量 API 可把多项编辑合并为一次候选重建；菜单
“批量添加 12 条示例断裂线”要求当前约束为空，在有效范围内生成互不相交的平行
断裂线并显示事务耗时。大数据单项编辑时仍会显示等待指针，后续将改为局部拓扑更新。

GRID 预览支持完整六参数仿射变换：通过三个目标角点映射最多 512×512 的 LOD 位图，
可显示超大、旋转或错切网格。显示缓存与源节点数解耦，生成等高线仍直接读取完整
GRID 句柄，不使用降采样高程替代原始计算。

v0.34 起，GUI 为当前显示 GRID 建立 128×128 像素、128 MiB 的二维 LOD 瓦片目录。
平移、滚轮缩放、拉框与适屏请求按视口中心优先加载；重叠视口共享已完成或正在加载的
瓦片。状态栏中的“瓦片LOD 源尺寸→输出尺寸 sN，复用/总数”分别表示缓存显示路径、
2 的幂次源节点采样间距和本次复用瓦片数量。切换高程/专题源时目录自动重建；分析、
等高线、转换和导出仍使用完整 GRID，不使用显示缓存近似值。

v0.35 起，打开或成功导出 `terrain.dgridb` 后，GUI 自动在同目录使用
`terrain.dgridb.dgtile` 稀疏旁车包。首次浏览的瓦片照常从 DGRIDB 生成并按需追加；关闭
程序再打开同一文件时，已浏览区域可直接复用。状态栏只有在本次结果实际读取了旁车负载
时才在复用计数后显示“（磁盘）”。旁车不可写、容量达到默认 2 GiB 或打开失败时自动
回退到 128 MiB 内存目录，缩放、平移、拉框和适屏仍正常工作。

`.dgtile` 不是权威地形数据，分析、转换、等高线和导出从不读取其中的近似像素。源
DGRIDB 变更后 GUI 会根据文件 revision 和源指纹自动重建过期旁车；坏头/目录或延迟发现
的坏瓦片也按策略重建。v0.36 可选择“数据交换 → 压缩当前 DGTILE 显示缓存”；程序取消
当前预览、原子保留最后有效瓦片，并在状态栏报告压缩前后 MiB、重复记录、损坏块和耗时。
Windows 会拒绝另一程序实例同时写同一路径。仍可在关闭程序后直接删除 `.dgtile` 彻底
释放空间，后续浏览按需重建。

v0.37 起，一个视口请求默认按 `s4 → s2 → 目标 LOD` 连续发布。GUI 不等待最终层完成，
而是每个 25 ms 轮询周期最多接收并绘制一层：先快速显示粗略地形，再替换中间层和精细层。
快速缩放或平移仍会取消过期任务并保留最近一幅完整画面；只有当前数据源、主题、窗口与
输出尺寸全部匹配的帧才能发布。状态栏中的 `sN` 会随渐进层变细，最终值与原单层请求
相同。此过程只改变显示缓存，不改变分析、转换、等高线或导出数据。

## 三维查看与漫游

单击“切换3D”后，程序对当前完整 TIN 做透视投影并按高程填色。三维视图只负责
查看；若单击查询、插入、删除或框选放大，程序会自动返回二维视图再执行操作。
v0.40 起优先使用 Direct3D 11 分块渲染、深度缓冲和视锥裁剪；状态栏的“GPU块
可见/总数，面 本帧/总数”可用于确认当前负载。硬件不可用时先尝试 WARP，仍失败才
显示“软件3D回退”并使用原抽样路径。

| 输入 | 三维操作 |
|---|---|
| 左键拖动 | 绕观察目标环视 |
| 右键或中键拖动 | 沿相机屏幕平面平移 |
| 鼠标滚轮 | 拉近或拉远相机 |
| `W/S` 或 `↑/↓` | 沿当前水平方向前进、后退 |
| `A/D` 或 `←/→` | 向左、向右漫游 |
| `Q/E` | 提升、降低观察目标 |
| `F` | 开启或关闭 TIN 高程贴地漫游 |
| `Ctrl+左键` | GPU 拾取并以橙色高亮三角形 |
| `+/-` | 增加、降低垂直夸张 |
| `Home` 或“全图” | 恢复三维适屏相机 |

三维模型会先减去数据中心并按 XY 范围归一化，再以双精度完成相机计算，因此可
直接查看大数值投影坐标。贴地开启时，WASD/方向键每次移动后用 DLL 精确定位 TIN
支撑三角形并插值高程；Q/E 手动升降仍有效。Ctrl+左键的 ID 通道只回读一个像素，
命中后状态栏显示三角形序号和中心世界坐标。

## 显示颜色

- 蓝—绿—黄—棕—白：由低到高的地形线框；
- 红色：编辑前被删除或受影响的三角形；
- 黄色：影响区域边界；
- 绿色：编辑后新增的三角形和边；
- 洋红色：查询点所在三角形；
- 白色圆点：查询点的最近网格顶点。
- GRID 图层：蓝—绿—黄—棕—白连续高程着色，青色外框；
- 等高线：黄色普通线，每五条抽样线使用浅色粗线增强层次。
- CDT：紫色域内网、青色外边界、洋红孔洞、橙色断裂线。
- 剖面：青色 A—B 定位线和端点，底部深色面板内为黄色高程曲线。
- 面积/土方：橙色边界、黄色顶点与半透明斜线填充，顶部深色面板显示量测结果。
- 双表面高差：蓝色填方、白色零差、红色挖方，右上角显示发散色带图例。
- 坡度/坡向：青色虚线支撑面、黄色查询点、青色下坡箭头和右上结果面板。

## 性能策略

DLL 始终精确查询所有与当前视口相交的三角形。为避免 GDI 阻塞，二维线框最多
抽样显示约45,000个普通 TIN 或 CDT 三角形；三维 Direct3D 主路径按16,384面分块并
提交视锥内全部块，不设18,000面抽样上限，只有软件回退仍使用该预算；GRID 只保留
不超过512×512的 LOD 显示位图；等高线绘制预算约20万个顶点。
这些限制只属于演示渲染器，不影响 DLL 数据、转换或文件导出。DGRIDB 已提供原始节点
和多级 LOD 的磁盘映射、视口预取、可取消异步请求以及二维空间瓦片目录、中心优先和
跨请求合并；v0.35 的 DGTILE 进一步提供跨会话热点瓦片复用，v0.37 已加入跨级渐进
发布，v0.40 已完成 Direct3D 分块渲染和拾取。超显存生产场景仍需增加三维块 LOD、
后台上传和显存 LRU。
双表面体积核心按行块并行且不要求 GUI 缓存整幅数据；只有请求高差专题时才额外
分配一幅同尺寸 double GRID。超大数据若只需统计，可在 DLL 调用中关闭高差输出。

## v1.0 交付与试运行建议

正式 SDK 的 `bin` 目录包含 GUI 与匹配的 DLL/运行时。先用 `sample_data` 完成 XYZ、GRID、
等高线和约束网的打开、互转、保存及 2D/3D 操作，再导入真实项目数据。百万点以上 TIN
应先适屏后框选局部编辑；大 GRID 应优先保存为 DGRIDB 并使用异步视口 LOD/DGTILE。
版本验收结果、硬件相关基线和已知边界见 `RELEASE_CHECKLIST.md`。
