# Windows GUI 演示说明

可执行文件：`build/dterrain_demo.exe`。

## 操作

| 控件 | 功能 |
|---|---|
| 模拟10万 | 生成约10万点的起伏地形并建网 |
| 模拟100万 | 生成约100万点，用于大数据交互演示 |
| 清空 | 清除当前三角网 |
| 查询模式 | 单击显示最近顶点和覆盖三角形 |
| 插入模式 | 单击插入一个按模拟地形函数计算高程的新点 |
| 删除模式 | 单击删除 XY 距离最近的顶点 |
| 框选放大 | 按住鼠标左键拖出矩形，松开后将选区放大并按画布比例适屏 |
| 全图 | 恢复到数据完整范围 |
| 导入XYZ | 导入 `.xyz/.txt/.csv` 散点并立即自动构网 |
| 保存网格 | 保存 `.dtmesh/.txt` 文本三角网或 `.dtin` 二进制网格 |
| 打开网格 | 打开并校验 `.dtmesh/.txt`，或加载 `.dtin` |
| 切换3D/切换2D | 在二维编辑视图与透视三维地形视图之间切换 |
| 高程×n | 依次选择 0.5、1.0、1.5、2.0、3.0、5.0、8.0 倍垂直夸张 |

“分析”菜单提供任意剖面以及多边形面积/土方量测，两类分析均可导出 CSV，并会
自动回到二维视图。

鼠标滚轮以指针位置为中心缩放。右键或中键拖动平移。在“框选放大”模式下，
按住左键拖出青色虚线选择框，松开后自动放大；选择过程中按 `Esc` 可取消。

XYZ 文件每个有效行包含 `x y z`，也接受逗号、分号和制表符；空行和以 `#`
开头的行会被忽略。文本三角网格式见 [TEXT_FORMATS.md](TEXT_FORMATS.md)。
便携目录附带的 `sample_data/sample_points.xyz` 可直接用于导入测试。

## 地形转换与图层

“地形转换”菜单提供以下自动化工作流：

| 命令 | 行为 |
|---|---|
| TIN → GRID | 按 TIN 范围生成最长边 401 节点、保持 XY 比例的规则 GRID |
| GRID → TIN | 以有效 GRID 节点重建普通 TIN；NoData 不自动形成硬孔洞，真实孔洞请使用独立 DCDT 约束网 |
| 从 TIN 生成等高线 | 扫描 TIN 高程范围并自动选取易读等高距 |
| 从 GRID 生成等高线 | 使用 GRID 有效高程范围自动选取等高距 |
| 等高线 → TIN | 以等高线折点和线高程重建普通 Delaunay；保留源等高线用于叠加检查 |
| 等高线 → GRID | 先由等高线折点建立临时 TIN，再生成最长边 401 节点 GRID |
| 从当前 TIN 创建约束网 | 复制普通 TIN 的全部顶点和 CRS，替换现有 CDT 基础点并清除旧约束 |
| 约束网 → GRID | 只插值有效域；外边界以外和孔洞内部写为 NoData |
| 从约束网生成等高线 | 只扫描有效域，等高线不会跨越外边界或孔洞 |

自动等高距使用 `1/2/5 × 10ⁿ` 级数，目标约 12 个高程级别。“图层”菜单可分别
显示或隐藏 TIN、GRID 高程着色和等高线；状态栏用 `T/G/C` 表示当前可见图层。
“全图”会计算所有可见二维图层的联合范围。

TIN 被重新生成、导入、打开或动态编辑后，已有 GRID 和等高线属于过期派生结果，
程序会自动释放这些图层。GRID→TIN 会保留源 GRID，清除不再匹配的等高线。
等高线→TIN 会保留作为输入的等高线、释放旧 GRID 和 CDT；等高线→GRID 保留源
等高线及现有 TIN，便于对照显示。

等高线反向转换属于近似重建：GUI 使用原折点，不会恢复等高线之间已经丢失的极值，
也不会强制等高线段成为普通 TIN 边。若输入只有一条直线或全部折点共线，DLL 会
拒绝转换并保留原数据。

## GRID、等高线与 GDAL 格式交换

“数据交换”菜单可独立导入和导出 `DGRID 1`、`DCONTOUR 1` 文本。导入 GRID 或
等高线不会强制删除现有 TIN，因此可以叠加检查。详细格式见
[TEXT_FORMATS.md](TEXT_FORMATS.md)。
安装目录的 `sample_data/sample_grid.dgrid` 和 `sample_contours.dcontour` 可直接
用于验证两个菜单入口。

“数据交换”菜单还提供“打开约束网 DCDT”和“保存约束网 DCDT”。打开后不替换
普通 TIN，可通过“图层→约束 Delaunay”独立显隐或叠加比较。安装目录的
`sample_data/sample_constraints.dcdt` 包含外边界、孔洞和断裂线，可直接验证。

使用 `DT_WITH_GDAL=ON` 构建时，“数据交换”菜单还提供：

| 命令 | 数据与行为 |
|---|---|
| 导入 GeoTIFF / COG | 读取第 1 波段为 GRID，只替换现有 GRID 图层 |
| 导出 GeoTIFF（DEFLATE） | 保存当前 GRID，使用 TILED、DEFLATE、BIGTIFF=IF_SAFER |
| 导出 Cloud Optimized GeoTIFF | 保存当前 GRID，使用 COG、DEFLATE、BIGTIFF=IF_SAFER |
| 导入 GeoPackage 等高线 | 读取 `elevation` 字段和 LineStringZ，只替换等高线图层 |
| 导出 GeoPackage 等高线 | 写入 `contours` 图层、`elevation/closed` 字段并建立空间索引 |

程序启动时分别检查 `GTiff`、`COG` 和 `GPKG` 驱动。普通构建或缺少相应驱动时，
五个菜单项按能力置灰，DGRID、DCONTOUR 和 DCDT 文本操作仍可使用。GDAL 格式
导入不会删除普通 TIN、CDT 或另一种派生图层，因此状态栏可继续显示 `T/G/C/D`
组合并进行叠加对照。导入失败时原图层保持不变。

GDAL 栅格像元中心对应本库 GRID 节点，读写时自动处理半像元偏移。当前 GUI 使用
第 1 波段，不执行坐标重投影；CRS WKT、仿射变换和 NoData 由 DLL 保存与恢复。

## 任意地形剖面分析

选择“分析→任意剖面（两次单击）”，在二维画布中先后单击 A、B 两点。程序沿
A—B 生成 401 个等距样本，绘制青色定位线和端点，并在画布底部显示高程曲线。
下一次单击会开始一条新剖面；第一次单击后按 `Esc` 可取消，菜单“清除剖面”可
清除已完成结果。

整条剖面固定使用一个数据源，选择优先级如下：

1. 当前可见且存在的 CDT 有效域；
2. 当前可见且存在的普通 TIN；
3. 当前可见且存在的 GRID；
4. 若以上均不可用，再按 CDT、TIN、GRID 顺序选择隐藏但存在的数据。

CDT 外边界以外或孔洞内、TIN 凸包外以及 GRID NoData 样本不会被其他图层补齐，
而是作为断点显示。这保证一条曲线具有一致的数据来源。状态栏和曲线面板显示平距、
有效样本数、最低/最高高程、净高差、累计上升、累计下降和相邻有效样本最大绝对
坡度。坡度以 `ΔZ / Δ平距 × 100%` 计算，断点两侧不跨段累计。

“分析→导出剖面 CSV”输出 UTF-8 文本，首行为数据源注释，字段包括
`index,distance,x,y,z,valid,grade_percent`。无效样本的 Z 和坡度留空；连续有效段
首点的坡度也留空。GRID 使用完整六参数仿射反算和双线性插值，并只读取每个样本
邻近的小窗口，适用于无法整体缓存的大栅格。

若剖面使用的源图层被重建、动态编辑、导入替换、转换替换或清除，程序会自动清除
旧剖面，避免把过期曲线误认为当前地形结果。

## 多边形面积与土方量测

选择“分析→面积/土方量测（逐点）”，在弹出的窗口中输入水平基准高程，然后在
二维画布中依次单击简单多边形的边界点。至少输入 3 点后按 `Enter` 闭合并计算；
`Backspace` 撤回草图末点，`Esc` 或“清除面积/土方量测”清除结果。自相交、重复边
或面积退化的边界会被拒绝。完成结果后再次单击会开始一块新的量测区域。

整块量测固定使用首点确定的数据源，优先级与剖面相同：可见 CDT、可见 TIN、
可见 GRID，其后回退到存在但隐藏的 CDT、TIN、GRID。CDT 外域/孔洞、TIN 凸包外
和 GRID NoData 不会从其他图层补齐。边界可跨越这些区域，结果会单独报告有效平面
面积和覆盖率。

结果含义如下：

| 字段 | 含义 |
|---|---|
| 多边形平面面积、周长 | 由输入 XY 边界直接计算 |
| 有效平面面积、覆盖率 | 当前数据源中可参与积分的区域及其占多边形面积比例 |
| 地表面积 | 对有效区域三维微三角形面积求和 |
| 最低/最高/平均高程 | 有效采样高程；平均值按平面面积加权 |
| 挖方 | 现状地形高于水平基准面的体积 |
| 填方 | 现状地形低于水平基准面的体积 |
| 净挖方 | 挖方减去填方，负值表示净填方 |

可用“分析→设置量测基准高程…”修改基准面并立即重算。“导出面积/土方 CSV…”输出
UTF-8 文本，前部为数据源、基准高程、各项统计和积分单元数，随后为
`index,x,y,z,valid` 边界顶点表。边界顶点无有效高程时 Z 留空；这不妨碍程序对
边界内部有效部分积分。

平面面积和周长是边界几何量；地表面积、高程与挖填方使用约 20,000 个微三角形
积分，是数值估算。孔洞/NoData 边缘按微三角形中心分类，误差受区域尺度、地形
起伏和数据分辨率影响。演示结果适合研究和方案比较，不应直接作为工程结算量。

若量测所用源图层被重建、动态编辑、导入替换、转换替换或清除，程序会自动清除
对应结果；修改无关图层不会使它失效。

## 坡度、坡向与地形法向

选择“分析→坡度/坡向分析（单击）”，程序自动回到二维视图。在地形上单击一次即
完成局部分析；之后每次单击会用新位置替换上次结果。按 `Esc` 或选择“清除坡度/
坡向结果”可清除覆盖层。

单击时按以下顺序固定选择一个数据源：可见 CDT、可见 TIN、可见 GRID，若均不可用
再按 CDT、TIN、GRID 回退到已存在但隐藏的数据。CDT 外域/孔洞、TIN 凸包外和
GRID 范围外/NoData 不会用其他图层在同一次调用中补值；当前选定源在查询点无效
时本次分析失败。可通过图层显隐明确选择叠加地形中的目标表面，结果面板会显示
最终数据源。

显示元素：

- 黄色圆点：实际查询 XY 和插值高程的平面位置；
- 青色虚线：TIN/CDT 的支撑三角形，或 GRID 的 2×2 支撑单元；
- 青色箭头：最大下降方向，箭头长度为屏幕显示长度，不代表实际坡长；
- 右上深色面板：数据源、Z、坡度角、坡向及八方位、`dz/dx`、`dz/dy` 和向上单位法向。

坡度相对水平面计量，水平面为 0°。坡向以地图 +Y 为北，顺时针表示下坡方向；
水平面显示“无唯一坡向”。TIN/CDT 在三角面内坡度恒定，查询落在边或顶点时采用
一个有效邻面；GRID 使用局部双线性导数，因此同一网格单元内数值可能随位置变化。
若当前还有完成的面积/土方面板，坡度面板自动下移，避免遮挡。

依赖的源图层发生动态编辑、重建、导入替换、转换替换或清除时，旧坡度结果自动
失效；无关图层变化不会清除它。

## 全幅地形专题图

“分析”菜单提供“生成全幅坡度专题图”“生成全幅坡向专题图”和“生成阴影地形图
（当前光照参数）”。已有 GRID 时直接分析；只有普通 TIN 或有效 CDT 时，程序先自动
按当前范围生成最长边 401 节点的高程 GRID，再计算专题结果。若两种三角网同时
存在，优先使用有效 CDT。

“设置专题分析与性能参数”依次设置高程单位系数 z-factor、太阳方位角、太阳高度角、
工作线程数和分块行数。
z-factor 默认 1.0，必须大于 0；例如 XY 为米而 Z 为毫米时应填 0.001。方位角以
北为 0°顺时针，任意输入会归一化到 0～360°；高度角范围为 -90～90°，默认光照
仍为 315°/45°。线程数填 0 自动、1 单线程或 2～64；块高填 0 使用默认 64 行，
也可填 1～1048576。通常保留自动设置；调试复现或宿主限制 CPU 时再显式调整。

三类专题图均保持原 GRID 的尺寸、旋转/错切仿射变换、CRS 和 NoData 区域：

| 专题图 | 显示与数值 |
|---|---|
| 坡度 | 绿色到紫色固定色带，0～60°，更陡坡度使用末端颜色 |
| 坡向 | 环形色相，按北→东→南→西→北覆盖 0～360°；水平面/无效区为 NoData |
| 阴影地形 | 0～255 灰度，默认西北 315°、高度角 45° 光照 |

右上角图例给出当前量纲和固定范围，因此切换不同数据集时颜色仍可比较。“恢复显示
高程 GRID”销毁当前专题结果并恢复蓝—绿—黄—棕—白高程色带；原高程 GRID 从未
被专题结果覆盖，剖面、量测、局部坡度查询和三维视图仍以它为数据源。

“导出当前专题图 DGRID”可在所有构建中使用；启用 GDAL/GTiff 后还可导出分块、
DEFLATE 压缩的 GeoTIFF。导出的数据是实际坡度角、坡向角或 0～255 阴影值，不是
屏幕 RGB 截图，可由 GIS 软件继续统计、分级或制图。

专题计算在后台任务中运行，状态栏持续显示百分比，窗口仍可重绘和移动。计算期间
按 `Esc` 或选择“取消正在进行的专题计算”可协作取消；此时不替换当前专题结果。
若计算期间关闭窗口，程序会先取消并等待工作线程安全退出，再完成关闭。普通菜单
命令在任务结束前会被忽略，避免源 GRID 被并发替换。

分块并行不会改变输出数值或额外复制一份全幅缓冲；工作线程直接填充结果 GRID 的
不同输出行。状态栏在完成后显示本次请求的线程模式与块高。小 GRID 或块数不足时，
DLL 会自动少用线程；过小块高可能增加调度开销，过大则降低负载均衡和取消响应。

## 约束 Delaunay 图层

DCDT 图层只绘制外边界以内、孔洞以外的有效域三角形。颜色含义：

- 紫色细线：域内约束 Delaunay 三角形；
- 青色粗线：地形外边界；
- 洋红粗线：孔洞边界；
- 橙色粗线：不改变域内外的断裂线。

状态栏图层码增加 `D`。全图范围会包含可见 CDT 的全部顶点。普通查询、插入和
删除按钮仍操作普通 TIN，不会隐式修改独立 CDT。

## 交互编辑约束

“约束编辑”菜单提供“绘制断裂线”“绘制外边界”“绘制孔洞边界”“批量添加 12 条
示例断裂线”“移动约束顶点”和“选择删除约束”。
建议先执行“地形转换→从当前 TIN 创建约束网”，或打开已有 DCDT。绘制操作：

1. 选择约束类型后，在二维画布中逐点单击；黄色草图显示当前输入；
2. 按 `Enter` 或选择“完成当前约束”提交；断裂线至少 2 点，边界至少 3 点；
3. 按 `Backspace` 撤销最后一点，按 `Esc` 或菜单命令取消整条草图；
4. 选择“选择删除约束”后，在青、洋红或橙色约束线 14 像素范围内单击；
5. 删除外边界前必须先删除依赖它的孔洞，否则 DLL 会拒绝操作并保持原网不变。

移动已有折点时，选择“移动约束顶点（两次单击）”：

1. 所有约束折点显示为白色小圆点；先在 14 像素范围内单击目标折点；
2. 被选折点显示为黄色大圆点，再单击新位置提交移动；
3. 按 `Esc` 或“取消当前约束”可取消选择；
4. 成功后约束 ID 保持不变，红色显示旧域面、黄色显示影响边界、绿色显示新域面和边；
5. 若新位置造成零长度边、未分段交叉或无效孔洞，提交被拒绝，原约束和拓扑保持不变。

安全删除单个折点时，选择“删除约束顶点（单击）”：

1. 白色小圆点表示可拾取的约束顶点，在 14 像素范围内单击目标点；
2. 普通顶点直接提交删除，并显示红/黄/绿影响结果；
3. 若该 XY 被多条约束共享，程序会显示引用数量并请求确认；选择“是”只从当前
   约束脱离，其他约束保持不变，选择“否”不改变数据；
4. 若该 XY 同时是基础地形点，删除的只是约束引用，基础点仍保留；
5. 删除会导致折线少于 2 点、闭合边界少于 3 点或产生非法拓扑时，DLL 拒绝操作并
   原子保留原网。整条约束删除仍使用“选择删除约束”。

草图点 Z 优先从当前 CDT 有效面插值，其次从普通 TIN 插值；两者均不可用时才使用
演示地形函数。约束提交或删除会释放已派生的 GRID/等高线，避免显示过期结果。
当前 DLL 对单条约束增删改仍执行候选 CDT 全量重建；移动/删除顶点为显示影响还会执行
完整的新旧域三角形差分。v0.11 的批量 API 可把多项编辑合并为一次候选重建；菜单
“批量添加 12 条示例断裂线”要求当前约束为空，在有效范围内生成互不相交的平行
断裂线并显示事务耗时。大数据单项编辑时仍会显示等待指针，后续将改为局部拓扑更新。

GRID 预览支持完整六参数仿射变换：通过三个目标角点映射位图，可显示旋转或错切
网格。超过 2000 万节点的 GRID 仍保留在 DLL 句柄中并可导出，但演示程序不缓存
高程着色，也不从该 GRID 直接生成等高线。

## 三维查看与漫游

单击“切换3D”后，程序对当前完整 TIN 做透视投影并按高程填色。三维视图只负责
查看；若单击查询、插入、删除或框选放大，程序会自动返回二维视图再执行操作。

| 输入 | 三维操作 |
|---|---|
| 左键拖动 | 绕观察目标环视 |
| 右键或中键拖动 | 沿相机屏幕平面平移 |
| 鼠标滚轮 | 拉近或拉远相机 |
| `W/S` 或 `↑/↓` | 沿当前水平方向前进、后退 |
| `A/D` 或 `←/→` | 向左、向右漫游 |
| `Q/E` | 提升、降低观察目标 |
| `+/-` | 增加、降低垂直夸张 |
| `Home` 或“全图” | 恢复三维适屏相机 |

三维模型会先减去数据中心并按 XY 范围归一化，再以双精度完成相机计算，因此可
直接查看大数值投影坐标。坐标轴采用红 X、绿 Y、蓝 Z。

## 显示颜色

- 蓝—绿—黄—棕—白：由低到高的地形线框；
- 红色：编辑前被删除或受影响的三角形；
- 黄色：影响区域边界；
- 绿色：编辑后新增的三角形和边；
- 洋红色：查询点所在三角形；
- 白色圆点：查询点的最近网格顶点。
- GRID 图层：蓝—绿—黄—棕—白连续高程着色，青色外框；
- 等高线：黄色普通线，每五条抽样线使用浅色粗线增强层次。
- CDT：紫色域内网、青色外边界、洋红孔洞、橙色断裂线。
- 剖面：青色 A—B 定位线和端点，底部深色面板内为黄色高程曲线。
- 面积/土方：橙色边界、黄色顶点与半透明斜线填充，顶部深色面板显示量测结果。
- 坡度/坡向：青色虚线支撑面、黄色查询点、青色下坡箭头和右上结果面板。

## 性能策略

DLL 始终精确查询所有与当前视口相交的三角形。为避免 GDI 阻塞，二维线框最多
抽样显示约45,000个普通 TIN 或 CDT 三角形；三维填充、深度排序和描边最多抽样显示约18,000个
三角形；GRID 预览缓存最多2000万个值并生成不超过512×512的显示位图；等高线
绘制预算约20万个顶点。此限制只属于演示渲染器，不影响 DLL 数据、转换或文件
导出。生产级千万点连续渲染宜改用 Direct3D/OpenGL、分块 LOD 和后台流式加载。
